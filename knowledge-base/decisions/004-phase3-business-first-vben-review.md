# 阶段 3 业务优先的 Vben 原生评审模式

日期：2026-07-15
状态：待用户评审
类型：设计与实施决策

## 背景

旧阶段 3 Demo 虽然可以交互，但没有证明与 Vben 兼容，且以通用后台对象替代了国网合同、客户回款、供应商采购和多源低质量数据的真实关系。用户评审后明确要求撤回该方向。

## 决策

1. 阶段 3 的评审页直接运行在 Vben 主应用中，不再用独立静态壳证明生产兼容性。
2. 页面主轴从成熟经营驾驶舱改为“合同档案与数据核验工作台”。
3. PDF 为法律源头真值；OCR DOCX、JSON、Excel 和飞书记录为派生或候选数据；人工核验后的系统记录逐步成为日常运营主数据。
4. 法律/交易形式与项目执行形态采用双轴分类。
5. 客户应收与供应商应付是方向相反的两条资金链，禁止用 `inbound` 或同一时间线合并表达。
6. Web 负责批量导入、全文核验、对账和配置；小程序负责轻量任务；App 负责弱网、离线和批量上传。三端共享 API、领域状态和设计令牌，不共享 Vben DOM 组件。
7. 默认视觉为浅色精密工作台；科技感来自证据轨道、关系线、状态反馈和数据时点，不来自大块深色背景或无业务依据的动效。

## 验收门槛

- 每个核心交互必须映射到现有 Vben/Ant Design Vue 组件并通过真实浏览器验证。
- 所有评审数字必须标注快照、来源和核验状态。
- 生产菜单、路由、后端和数据迁移必须在用户确认业务模型后分批实施。
