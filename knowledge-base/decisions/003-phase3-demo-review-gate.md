# 阶段 3 Demo 评审门

日期：2026-07-15

状态：部分保留，视觉与载体已被 004 决策取代
类型：设计实施决策

> “用户评审后再生产改造”的门禁继续有效；独立静态 Demo、深海蓝大色域和旧履约链方向已失效，当前评审改为 Vben 原生浅色业务工作台，见 `004-phase3-business-first-vben-review.md`。

## 背景

阶段 3 同时涉及菜单树、规范路由、全局壳层、合同与项目任务流、客户应收和供应商履约。
如果在视觉与任务流未确认前直接改造生产路由和后端菜单，会扩大返工范围，也容易让 Demo 验收被误认为生产完成。

## 决策

阶段 3 使用“设计基线 → 独立高保真 Demo → 用户评审 → 生产纵向切片”的评审门：

1. 独立 Demo 使用脱敏模拟数据，不连接生产 API。
2. Demo 必须覆盖规范菜单、核心任务流、页面状态、响应式和无障碍基础项。
3. 浏览器验收通过只证明 Demo 可评审，不改变阶段 3 的生产完成状态。
4. 用户确认菜单、视觉、履约链和返回流后，才进入生产路由与后端改造。
5. 生产先完成合同台账→合同详情→项目详情→返回的纵向切片，再扩展客户应收和供应商履约。

## 视觉实施原则

- 使用“安静的未来指挥台”方向：深海蓝导航、云雾白内容、电光青蓝状态。
- 把视觉大胆度集中在“履约链”，其余页面保持高信息密度与可扫描性。
- 科技感来自业务轨道、数据时点、等宽编号和短时反馈，不使用持续粒子和全屏霓虹。
- 支持移动端纵向履约链、键盘焦点和 reduced-motion。

## 后端边界

Demo 可使用现有字段结构；生产实现前必须先解决：

- 项目正式菜单和规范路由。
- 供应商详情自重定向。
- 详情来源返回契约。
- 供应商付款接口地址。
- 列表消费 Query 和提醒单次导航。

## 影响

### 正向影响

- 设计评审和生产改造边界清晰。
- 可以在零生产风险下验证视觉与任务流。
- 后端改造围绕已确认的纵向切片展开。
- 验收证据可以区分 Demo、源码、运行时和部署版本。

### 成本

- 需要维护一个短期静态 Demo。
- 用户评审成为生产实现的显式前置条件。

## 退出条件

用户确认四任务域、驾驶舱四视图、“履约链”、页面信息密度和明暗主题范围后，本决策进入“已接受”，阶段 3 转入生产批次 A。
