# 阶段 3 任务导向信息架构

日期：2026-07-15

状态：已被 004 决策取代
类型：设计基线决策

> 本文保留为第一轮设计历史。用户评审后，任务域、履约链和视觉方向均降级为待验证假设；当前决策见 `004-phase3-business-first-vben-review.md`。

## 背景

当前系统按开发模块拆分为概览、合同管理、客户财务、供应商管理和供应商财务。
项目执行没有正式菜单入口，供应商档案与供应商财务被拆开，顶部菜单、左侧菜单和标签栏同时承担导航。

设计审计结论为 REDESIGN。
阶段 3 必须先重做信息架构、导航和主流程，不能只更换颜色和卡片样式。

## 决策

目标信息架构按用户任务收敛为四个任务域：

1. 驾驶舱。
2. 合同履约。
3. 客户应收。
4. 供应商履约。

桌面端只保留左侧栏作为全局业务导航。
顶栏只保留工具，关闭全局页面标签栏，面包屑用于表达位置。

驾驶舱保留单路由四视图。
项目执行归入合同履约。
供应商档案、发票和付款归入供应商履约。

详情页全部隐藏，通过规范列表、关联对象和来源优先返回契约连接。

## 原因

- 匹配项目总监“判断、跟进、核对”的工作心智。
- 降低从六类对象中查找入口的认知成本。
- 让合同与项目、发票与回款、供应商与付款形成连续任务。
- 让每个业务对象只有一个规范入口和返回目标。
- 保留现有 Vben、Vue、Ant Design Vue 和 FastAPI 技术栈。

## 影响

### 正向影响

- 项目执行获得正式入口。
- 供应商信息和供应商财务不再割裂。
- 主要任务不再依赖隐藏旧页面。
- 菜单树、路由注册表、页面标题和返回契约可以使用同一词汇。

### 迁移成本

- 需要建立新的父级路径和旧路径单向重定向。
- 需要更新页面内命名路由、硬编码路径和 E2E。
- 需要关闭 mixed-nav 和全局标签栏。
- 需要在用户首击测试后确认菜单命名。

## 约束

- 子路由必须使用相对路径。
- 后端菜单是业务路由唯一真值。
- 旧路径只允许单向重定向，计划保留到 2026-08-15。
- 在首击测试和纵向切片通过前，本决策保持“提议”状态。
- 不在本决策中新增角色权限、业务模块或数据自动修复。
