## 一、项目类型识别

### 1.1 科研类项目识别关键词

**项目名称关键词**：
- "研究"、"研发"、"科技项目"、"科研项目"
- "课题"、"关键技术"、"核心技术研究"
- "技术创新"、"方法研究"、"应用研究"

**合同特征**：
- 包含"研究内容"、"考核目标"、"项目进度安排"章节
- 包含"项目支付计划"、"经费预算"章节
- 研究周期通常1-3年

### 1.2 服务类项目识别关键词

**项目名称关键词**：
- "服务"、"运维"、"维护"、"支撑"
- "实施"、"建设"、"改造"、"升级"
- "供应"、"采购"、"采购服务"

**合同特征**：
- 包含"服务内容"、"服务要求"、"交付物"章节
- 包含"工期"、"完成时间"、"验收标准"章节
- 服务周期通常较短（月/季度/年）

## 二、科研类项目识别规则

### 2.1 研究阶段识别

**识别目标**：
- 提取所有研究阶段（N个）
- 每个阶段包含：阶段名称、研究内容、考核目标、时间节点

**章节关键词**：
- "项目进度安排"、"研究计划"、"实施计划"
- "项目实施方案"、"研究方案"

**阶段识别规则**：

```
规则1：章节编号 + 阶段名称
  示例：
  "一、项目进度安排"
  "1. 第一阶段：需求分析"
  "2. 第二阶段：系统设计"
  "3. 第三阶段：系统开发"

规则2：时间节点 + 阶段名称
  示例：
  "2024年1月-3月：需求分析阶段"
  "2024年4月-6月：系统设计阶段"
  "2024年7月-12月：系统开发阶段"

规则3：阶段名称 + 内容描述
  示例：
  "第一阶段：完成项目研究实施方案"
    研究内容：编写项目研究实施方案
    考核目标：通过专家评审
```

**字段提取规则**：

| 字段 | 识别规则 | 示例 |
|------|---------|------|
| **阶段名称** | "第X阶段"、"阶段X"、时间范围 | "第一阶段"、"2024年1月-3月" |
| **研究内容** | "完成..."、"开展..."、"研究..." | "完成项目研究实施方案" |
| **考核目标** | "考核"、"验收"、"交付"、"成果" | "通过专家评审"、"提交实施方案" |
| **时间节点** | 时间格式（年月日） | "2024年3月"、"2024年3月31日前" |

### 2.2 付款计划识别

**识别目标**：
- 提取所有付款节点
- 建立付款与研究阶段的对应关系

**章节关键词**：
- "项目支付计划"、"经费预算"、"付款安排"
- "支付计划"、"经费支付"

**付款识别规则**：

```
规则1：年度 + 付款条件 + 金额
  示例：
  "2024年  完成项目研究实施方案  18.03万元"

规则2：阶段 + 付款条件 + 金额
  示例：
  "第一阶段完成后支付合同总额的30%，即18.03万元"

规则3：时间节点 + 付款条件 + 金额
  示例：
  "合同签订后15日内支付合同总额的30%"
  "项目验收合格后支付剩余款项"
```

**字段提取规则**：

| 字段 | 识别规则 | 示例 |
|------|---------|------|
| **付款条件** | "完成..."、"验收..."、"签订..." | "完成项目研究实施方案" |
| **金额** | 数字 + 单位（万元、元） | "18.03万元"、"180300元" |
| **付款时间** | 年度、时间节点 | "2024年"、"合同签订后15日内" |
| **对应阶段** | 从付款条件中推断 | "完成项目研究实施方案" → 第一阶段 |

### 2.3 阶段与付款对应关系

**对应关系类型**：

| 类型 | 说明 | 示例 |
|------|------|------|
| **1:1** | 1个阶段对应1笔付款 | 第一阶段完成 → 支付18.03万元 |
| **N:1** | 多个阶段对应1笔付款 | 第一、二阶段完成 → 支付36.06万元 |
| **1:N** | 1个阶段对应多笔付款（少见） | 第一阶段完成 → 分两笔支付 |

**推断规则**：

```
规则1：付款条件明确提及阶段
  "完成第一阶段研究内容后支付..."
  → 明确对应：第一阶段 → 该笔付款

规则2：付款条件与研究内容匹配
  付款条件："完成项目研究实施方案"
  研究阶段："第一阶段：完成项目研究实施方案"
  → 推断对应：第一阶段 → 该笔付款

规则3：时间节点匹配
  付款时间："2024年3月"
  研究阶段："2024年1月-3月：需求分析阶段"
  → 推断对应：需求分析阶段 → 该笔付款

规则4：多阶段合并付款
  付款条件："完成第一阶段和第二阶段研究内容后支付..."
  → 明确对应：第一阶段+第二阶段 → 该笔付款
```

## 三、服务类项目识别规则

### 3.1 项目工期识别

**识别目标**：
- 提取项目开始时间、结束时间、总工期
- 提取关键时间节点

**章节关键词**：
- "工期"、"服务期限"、"合同期限"
- "项目周期"、"实施周期"

**时间识别规则**：

```
规则1：明确工期天数
  "项目工期：90天"
  "自合同签订之日起90日内完成"

规则2：时间范围
  "2024年1月1日至2024年3月31日"
  "合同期限：2024年1月1日-2024年12月31日"

规则3：截止日期
  "在2024年6月30日前完成"
  "2024年12月31日之前交付"

规则4：相对时间
  "合同签订后X日内完成"
  "收到通知后X个工作日内完成"
```

**字段提取规则**：

| 字段 | 识别规则 | 示例 |
|------|---------|------|
| **开始时间** | "自...起"、"从...开始" | "自合同签订之日起" |
| **结束时间** | "至...止"、"在...之前完成" | "2024年6月30日前完成" |
| **总工期** | 数字 + 单位（天、月、年） | "90天"、"12个月" |
| **关键节点** | 里程碑时间 | "中期验收"、"最终验收" |

### 3.2 服务要求识别

**识别目标**：
- 提取服务内容、交付物、验收标准

**章节关键词**：
- "服务内容"、"服务要求"、"服务标准"
- "交付物"、"验收标准"、"质量要求"

**字段提取规则**：

| 字段 | 识别规则 | 示例 |
|------|---------|------|
| **服务内容** | "提供..."、"开展..."、"实施..." | "提供系统运维服务" |
| **交付物** | "提交..."、"交付..."、"提交成果" | "提交运维报告"、"交付系统" |
| **验收标准** | "验收..."、"质量标准..." | "符合XX标准"、"通过专家评审" |

## 四、识别流程

### 4.1 科研类项目识别流程

```
步骤1：识别项目类型
  - 检测项目名称关键词
  - 检测章节关键词

步骤2：提取研究阶段
  - 定位"项目进度安排"章节
  - 提取所有阶段信息（阶段名称、内容、考核、时间）
  - 存储为阶段列表

步骤3：提取付款计划
  - 定位"项目支付计划"章节
  - 提取所有付款信息（条件、金额、时间）
  - 存储为付款列表

步骤4：建立对应关系
  - 分析付款条件与研究内容
  - 推断阶段与付款的对应关系
  - 标注对应关系类型（1:1或N:1）

步骤5：输出结果
  - 研究阶段列表（N个阶段）
  - 付款计划列表（M笔付款）
  - 对应关系矩阵
```

### 4.2 服务类项目识别流程

```
步骤1：识别项目类型
  - 检测项目名称关键词
  - 检测章节关键词

步骤2：提取工期信息
  - 定位"工期"章节
  - 提取开始时间、结束时间、总工期
  - 提取关键时间节点

步骤3：提取服务要求
  - 定位"服务内容"章节
  - 提取服务内容、交付物、验收标准

步骤4：输出结果
  - 工期信息（开始、结束、总工期）
  - 服务要求列表
  - 关键节点列表
```

## 六、注意事项

### 6.1 日期格式标准化

| 原格式 | 标准格式 |
|--------|----------|
| 2024年1月 | 2024-01 |
| 2024年1月1日 | 2024-01-01 |
| 24年1月1日 | 2024-01-01 |
| 2024.1.1 | 2024-01-01 |
| 2024-1-1 | 2024-01-01 |

### 6.2 金额格式标准化

| 原格式 | 标准格式 |
|--------|----------|
| 18.03万元 | 180300元 |
| 180300元 | 180300元 |
| 壹拾捌万零叁佰元 | 180300元 |
| 18.03万 | 180300元 |

### 6.3 阶段与付款对应关系推断优先级

1. **优先级1**：付款条件明确提及阶段（最准确）
2. **优先级2**：付款条件与研究内容匹配（较准确）
3. **优先级3**：时间节点匹配（需人工确认）
4. **优先级4**：无法确定（标注为"待人工确认"）
