# 表格识别方案研究 - 付款计划提取

> **研究日期**：2026-06-24  
> **研究目标**：付款计划表格识别  
> **当前问题**：V2版提取的付款计划金额不准确（部分为0.0）

---

## 一、当前问题分析

### 1.1 实际付款计划表格（第12-13页）

**原始表格结构**：
```
项目支付计划
年度    收款单位                  付款条件              甲方提供经费（万元）  乙方自筹经费（万元）
2024年  北京振华永创智能科技有限公司  完成项目研究实施方案   18.03                0
2024年  北京振华永创智能科技有限公司  完成1份专利申请        18.03                0
2024年  北京振华永创智能科技有限公司  完成2篇论文录用和1份... 9.02                0
2025年  北京振华永创智能科技有限公司  完成1篇论文录用        12.14                0
2025年  北京振华永创智能科技有限公司  完成软件著作权登记     12.14                0
2025年  北京振华永创智能科技有限公司  完成验收              6.08                0
```

**表格特点**：
1. **跨多行**：每个单元格内容可能跨多行
2. **列对齐不严格**：OCR识别后列对齐被打乱
3. **嵌套结构**：表头和表格内容混在一起

### 1.2 当前提取结果

**V2版提取结果**：
```
第1笔: {'金额（万元）': 18.03}  ✅ 正确
第2笔: {'金额（万元）': 0.0}   ❌ 错误（应为18.03）
第3笔: {'金额（万元）': 18.03}  ✅ 正确
第4笔: {'金额（万元）': 0.0}   ❌ 错误（应为9.02）
第5笔: {'金额（万元）': 9.02}   ✅ 正确
第6笔: {'金额（万元）': 0.0}   ❌ 错误（应为12.14或6.08）
```

**问题根因**：
- 表格内容跨多行，OCR识别后分行
- 当前列识别逻辑没有考虑跨行情况
- "乙方自筹经费"列的0值被误识别为"甲方提供经费"

## 二、表格识别方案研究

### 2.1 方案对比

| 方案 | 优势 | 劣势 | 适用性 |
|------|------|------|--------|
| **PaddleOCR PP-StructureV3** | 官方支持，表格识别专用 | 需要额外安装，模型较大 | ✅ 推荐 |
| **Table Transformer (TATR)** | 开源，准确率高 | 需要单独部署 | ⚠️ 复杂 |
| **自定义规则解析** | 轻量，可控 | 规则复杂，通用性差 | ⚠️ 维护成本高 |
| **Camelot/Tabula** | PDF表格专用 | 仅适用于结构化PDF | ❌ 不适用于OCR结果 |

### 2.2 推荐方案：PaddleOCR PP-StructureV3

#### 方案概述

**PP-StructureV3**是PaddleOCR官方的文档结构分析工具，专为表格识别设计。

**核心功能**：
1. **表格检测**：自动检测文档中的表格区域
2. **表格结构识别**：识别表格的行、列结构
3. **表格内容识别**：结合OCR识别表格内容
4. **输出结构化数据**：输出JSON格式的表格数据

#### 技术架构

```
PDF图像 → 表格检测 → 表格结构识别 → 单元格识别 → 结构化输出
         (检测模型)    (结构模型)      (OCR模型)
```

#### 安装方法

```bash
# 安装PaddleOCR完整版（包含PP-Structure）
pip install paddleocr[all]

# 或单独安装PP-Structure
pip install paddleocr
pip install paddlestructure
```

#### 使用示例

```python
from paddleocr import PPStructure

# 初始化PP-Structure
table_engine = PPStructure(show_log=True, use_angle_cls=True, recovery=True)

# 识别PDF中的表格
result = table_engine(pdf_path)

# 输出表格结构
for page_result in result:
    for table_result in page_result:
        if table_result['type'] == 'table':
            # 表格HTML
            html = table_result['res']['html']
            # 表格JSON
            cells = table_result['res']['cells']
```

#### 预期效果

**优势**：
- ✅ 自动识别表格结构
- ✅ 处理跨行跨列单元格
- ✅ 输出结构化数据
- ✅ 准确率高（官方数据95%+）

**劣势**：
- ⚠️ 需要额外安装（约500MB）
- ⚠️ 识别速度较慢（比纯OCR慢2-3倍）
- ⚠️ 需要GPU加速（CPU也可以，但更慢）

### 2.3 备选方案：自定义规则解析

如果不想安装PP-Structure，可以使用自定义规则解析。

#### 核心思路

**方法**：基于"年度"关键词定位行，然后解析每行的列

```python
def parse_payment_table(text: str) -> List[Dict]:
    """解析付款计划表格"""
    
    payments = []
    lines = text.split('\n')
    
    # 定位表格开始（"项目支付计划"章节）
    table_start = None
    for i, line in enumerate(lines):
        if '项目支付计划' in line or '年度' in line and '收款单位' in line:
            table_start = i
            break
    
    if not table_start:
        return []
    
    # 解析表格行
    current_payment = {}
    for i in range(table_start, min(table_start + 50, len(lines))):
        line = lines[i].strip()
        
        # 检测年度（新行开始）
        if re.match(r'20\d{2}年', line):
            # 保存上一条记录
            if current_payment:
                payments.append(current_payment)
            
            # 开始新记录
            current_payment = {'年度': line}
        
        # 检测金额（数字+可能的小数）
        elif re.match(r'\d+\.?\d*', line):
            amount = float(line)
            # 判断是"甲方提供经费"还是"乙方自筹经费"
            # 根据上下文判断（需要更复杂的逻辑）
            if '金额（万元）' not in current_payment:
                current_payment['金额（万元）'] = amount
            else:
                current_payment['乙方自筹（万元）'] = amount
        
        # 检测收款单位
        elif '公司' in line and '有限' in line:
            current_payment['收款单位'] = line
        
        # 检测付款条件
        elif '完成' in line or '验收' in line:
            current_payment['付款条件'] = line
    
    # 保存最后一条记录
    if current_payment:
        payments.append(current_payment)
    
    return payments
```

#### 优势与劣势

**优势**：
- ✅ 轻量级，无需额外安装
- ✅ 可控性强，可针对特定格式优化
- ✅ 速度快

**劣势**：
- ❌ 通用性差，每种表格格式都需要编写规则
- ❌ 维护成本高
- ❌ 容错性差，OCR识别错误会影响解析
