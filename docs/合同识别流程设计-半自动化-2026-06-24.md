## 一、识别方式选择

### 1.1 暂停PP-StructureV3的原因

**技术问题**：
- 识别大文件（>5MB）导致内存不足
- 识别速度慢（约2-3分钟/合同）
- 识别准确率有限（约83%）

**优先级问题**：
- 先解决表格设计和数据流转逻辑
- 再优化识别技术

---

### 1.2 采用"人工识别+AI辅助"方式

**原因**：
- ✅ 数据100%完整（人工确认）
- ✅ 准确率高（人工+AI双重验证）
- ✅ 成本可控（效率与质量平衡）

---

## 二、识别流程设计

### 2.1 完整流程（6步骤）

**步骤1：下载合同PDF**
```
输入：合同文件token
处理：从飞书云文档下载PDF
输出：本地PDF文件
```

**步骤2：人工识别关键字段**
```
输入：合同PDF文件
处理：人工阅读并填写关键字段清单
输出：结构化字段数据（JSON格式）

关键字段清单（必填）：
1. 合同编号
2. 项目名称
3. 甲方全称
4. 乙方全称
5. 签订日期
6. 合同金额（含税）
7. 不含税金额
8. 税率
9. 税额
10. 付款计划（阶段、时间、金额）
11. 交付物清单（阶段、名称、类型、数量、时间）
12. 项目简介
13. 甲方联系人
14. 甲方联系人电话
15. 预计交付日期
16. 合同类型
17. 技术方向
```

**步骤3：AI验证字段**
```
输入：结构化字段数据
处理：AI验证字段完整性和准确性
输出：验证报告（缺失字段、异常字段）
```

**步骤4：填充数据到表格**
```
输入：验证后的字段数据
处理：按顺序填充到各表格
  1. 填充合同信息表（档案表）
  2. 填充项目管理表（主表）
  3. 分解付款计划 → 填充回款跟踪表
  4. 分解交付物清单 → 填充交付物清单表
输出：填充后的记录ID
```

**步骤5：建立关联关系**
```
输入：各表格记录ID
处理：建立关联关系
  - 项目管理表 ↔ 合同信息表（1对1）
  - 项目管理表 ↔ 回款跟踪表（1对多）
  - 项目管理表 ↔ 交付物清单表（1对多）
输出：关联关系建立完成
```

**步骤6：数据验证**
```
输入：填充后的表格数据
处理：验证完整性和一致性
  - 字段完整性检查
  - 数据一致性检查
  - 关联关系检查
输出：验证报告
```

---

### 2.2 数据填充顺序

**第一优先级**：合同信息表（档案表）
- 存储合同原始信息
- 作为数据源

**第二优先级**：项目管理表（主表）
- 从合同信息表提取关键字段
- 填充项目简介、甲方联系人、预计交付日期等

**第三优先级**：回款跟踪表（从表）
- 从合同信息表的付款计划分解
- 填充阶段、计划金额、计划时间

**第四优先级**：交付物清单表（从表）
- 从合同信息表的交付物清单分解
- 填充阶段、交付物名称、类型、数量、时间

## 三、字段提取规则库

### 3.1 项目管理表字段提取规则

| 字段 | 提取位置 | 提取方式 | 验证规则 |
|------|---------|---------|----------|
| 项目名称 | 合同首页 | 标题识别 | 非空，长度≥5 |
| 项目简介 | 合同正文 | "项目概述"或"技术要求"章节 | 非空 |
| 甲方联系人 | 合同签署页 | "联系人"或"项目负责人" | 非空 |
| 甲方联系人电话 | 合同签署页 | 联系电话 | 11位手机号或固话 |
| 预计交付日期 | 合同正文 | 最后交付阶段的日期 | 日期格式 |
| 交付物清单 | 合同正文 | "交付物"章节 | 非空 |
| 合同类型 | 合同标题 | 标题关键词识别 | 科技项目/技术服务/工程服务 |
| 技术方向 | 合同正文 | 交付内容技术识别 | 人工智能/大数据/物联网等 |

---

### 3.2 付款计划提取规则

**格式要求**：
```
阶段1：时间（YYYY-MM-DD）：金额（元）
阶段2：时间（YYYY-MM-DD）：金额（元）
...
```

**提取位置**：合同正文"付款方式"或"付款条款"章节

**分解规则**：
1. 识别阶段序号（第1阶段、首付款等）
2. 识别付款条件（合同签订后、验收后等）
3. 识别付款时间（具体日期或相对时间）
4. 识别付款金额（具体金额或比例）

---

### 3.3 交付物清单提取规则

**格式要求**：
```
阶段1：
  - 交付物1：类型：数量：时间
  - 交付物2：类型：数量：时间
阶段2：
  - 交付物1：类型：数量：时间
...
```

**提取位置**：合同正文"交付物"或"交付内容"章节

**分解规则**：
1. 识别阶段序号
2. 识别交付物名称
3. 识别交付物类型（论文、专利、软著、报告等）
4. 识别数量
5. 识别交付时间

## 四、异常处理机制

### 4.1 字段缺失处理

**必填字段缺失**：
- 暂停填充
- 提示用户补充
- 记录到异常日志

**可选字段缺失**：
- 继续填充
- 标记为"待补充"
- 后续补充

---

### 4.2 数据冲突处理

**冲突类型**：
1. 合同信息与项目管理表不一致
2. 付款计划金额与合同金额不符
3. 交付时间与预计交付日期矛盾

**处理方式**：
- 记录冲突详情
- 提示用户确认
- 保留确认后的数据
