# 供应商详情页改造方案

> 改造目标：聚焦**业务协作数据**，将供应商详情从"基本信息展示"升级为"合作关系总览"

---

## 一、当前问题

| 问题 | 说明 |
|------|------|
| 信息散乱 | 发票、付款、合同三个卡片并列，缺乏关联性 |
| 合作项目不突出 | 无法一眼看出与该供应商有哪些项目合作、进度如何 |
| 财务数据孤立 | 看不到每个项目的合同金额、已开票、已付款的对比 |
| 联系人信息薄弱 | 仅一个联系人+电话字段，缺乏多联系人支持 |
| 导航缺失 | 页面内容多但没有锚点导航，用户需手动滚动 |

## 三、改造要点

### 3.1 新增功能

- **吸顶锚点导航** — 点击跳转到对应区块，固定在页面顶部
- **合作项目卡片** — 每个项目独立卡片展示，含进度条（开票率/付款率），可点击跳转到合同详情
- **多联系人管理** — 支持多个联系人（姓名、职位、电话、邮箱），可增删改
- **项目财务对比** — 每个项目显示合同额 vs 已开票 vs 已付款的对比进度条

### 3.2 增强功能

- **KPI 指标** — 增加"合作项目数"、"合同履约率"等指标
- **发票/付款表格** — 从简单列表升级为完整 VxeTable，支持排序、筛选、分页
- **状态颜色体系** — 统一的财务状态颜色（合同金额=灰色，已开票=蓝色，已付款=绿色，未付款=橙色）

### 3.3 后端改动

- `GET /api/suppliers/{id}` 保持现有返回结构
- 新增：`GET /api/suppliers/{id}/contacts` — 多联系人接口
- 新增：`POST/PUT/DELETE /api/suppliers/{id}/contacts` — 联系人 CRUD

### 3.4 数据库改动

- 新建 `supplier_contacts` 表（id, supplier_id, name, position, phone, email, is_primary, notes）

## 四、优先级规划

| 优先级 | 内容 | 工作量 |
|--------|------|--------|
| P0 | 吸顶导航 + 区块重构（不改数据） | 小 |
| P0 | 合作项目卡片 + 进度条 | 中 |
| P0 | 发票/付款升级为 VxeTable | 中 |
| P1 | 多联系人管理（含后端接口） | 中 |
| P1 | 企查查区块整合 | 小 |
| P2 | 项目财务对比图表 | 大 |
