## 任务2: 合同详情页 UI 重设计
**现状**: 当前布局混乱，信息密度过高，科研/服务类未区分。
**目标**: 轻重结合、视觉层次分明、两种合同类型用不同模板。

### 2A. 科研类详情模板
```
┌─ 基本信息（合同编号/名称/类型/金额/时间/甲乙方）────────┐
├─ 研究阶段详情（列表）                                    │
│   每个阶段: [研究阶段5.1] 日期范围 → 展开后可看内容/考核   │
├─ 经费预算（表格，仅科研类显示）                          │
├─ 付款进度（与阶段关联显示）                              │
├─ 项目团队                                              │
├─ 合同条款（默认折叠）                                    │
└─ 源文件                                                ┘
```

### 2B. 技术服务类详情模板
```
┌─ 基本信息 ───────────────────────────────────────────────┐
├─ 服务内容 / 服务方式 / 服务地点 / 服务期限              │
│   服务进度 / 服务质量要求（6 个信息行）                    │
├─ 付款进度                                              │
├─ 项目团队                                              │
├─ 合同条款（默认折叠）                                    │
└─ 源文件                                                ┘
```

### 2C. UI 规范
- 颜色：简洁专业（蓝/灰主色调，状态用语义色）
- 字体：统一字号层级，不做花哨
- 间距：统一卡片间距，不过度稀疏
- 折叠：所有可折叠区域默认关闭
- 响应式：桌面端为主，移动端适配
- 不做 Three.js/玻璃拟态等炫技效果——实用优先
