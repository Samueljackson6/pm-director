## 2. P0 风险

### 2.1 供应商详情 GET 可能触发数据库初始化

`backend.qcc_sync.get_local_qcc_data()` 当前在读取前调用 `init_qcc_tables()`；该函数包含 `CREATE TABLE IF NOT EXISTS` 和 `commit()`。即使现有表存在，接口契约仍不是严格只读。

处理要求：

- 将迁移/建表从读取函数移到显式初始化或迁移流程。
- 为供应商详情 GET 建立无 DDL、无 DML、无 commit 的测试或数据库 trace。
- 修复前不能宣称 G003/G006 已满足数据库零变更。

### 2.2 业务数据库校验未最终闭环

已记录的接管基线：

- 文件大小：`1,736,704` 字节。
- mtime（UTC）：`2026-07-17T01:56:07Z`。
- 基线 SHA256：`8651DC7EA351EBCA9FC2D6E718B04A70A373759AEB9795C709CBA6F1DA154849`。

当前大小和 mtime 与基线一致，但数据库被运行服务占用时无法重新计算哈希。接手团队应在经过授权且可安全停止占用后复核；不得用更早备份覆盖当前数据库。

### 2.3 仓库存在疑似明文凭据

只读审计曾在根 `MEMORY.md` 和企查查集成文档附近发现疑似 OCR/第三方凭据。交接提交不会复述这些值。

处理要求：

- 在合并或公开仓库前完成秘密扫描。
- 确认后立即轮换凭据、从工作树和历史中清除，并改用环境变量或配置中心。
- 禁止在 Issue、PR、日志、截图和交接文档中复制真实值。

## 6. 测试注意事项

- 客户发票和供应商详情的源码扫描型 Vitest 使用 `fileURLToPath(import.meta.url)`，必须加 `--environment node`。
- 默认浏览器模拟环境会在测试收集阶段报“URL must be of scheme file”，这不是业务用例失败。
- 当前 pnpm 会提示根 `package.json` 的部分 `pnpm` 字段迁移警告；需单独评估，不应与阶段 3 页面回归混淆。
- 2026-07-19 应用全量 `vue-tsc` 报告 511 条错误，主要来自资产、BPM、WMS 等既有模块，同时包含驾驶舱测试对象缺少 `project_execution` 等契约问题。接手团队应先保存基线、区分新增与既有错误，再按模块治理。
- 直接 Vite 生产构建已经完成，但构建输出仍包含循环分块和大体积分块警告；这些警告属于 G006 性能与发布验收范围。
- G002 的浏览器和构建证据不能替代 G003 之后的最终回归；核心页面继续变化后必须重跑。
