## 2. 新增表：`contract_clauses`（条款级）

```sql
CREATE TABLE contract_clauses (
  clause_id      TEXT PRIMARY KEY,
  contract_id    TEXT NOT NULL,
  clause_category TEXT NOT NULL,   -- breach_liability|liquidated_damages|penalty|overdue|compensation|confidentiality|ip|force_majeure|termination
  trigger_type   TEXT,             -- 逾期交付|验收不通过|保密违约|解除合同|其他
  rate_pct       REAL,             -- 违约金/罚款比例（占报酬 %）
  ld_base        TEXT DEFAULT '技术服务报酬',  -- 计算基数
  threshold_days INTEGER,          -- 逾期解除阈值（N 日）
  refund_full    BOOLEAN DEFAULT 0,-- 是否退还全部款项
  clause_text    TEXT,             -- 原文片段（留痕，满足审计）
  source_doc     TEXT,             -- 源 DOCX 文件名
  extracted_at   TEXT,
  created_at     TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (contract_id) REFERENCES contracts(contract_id)
);
CREATE INDEX idx_cc_contract ON contract_clauses(contract_id);
CREATE INDEX idx_cc_cat ON contract_clauses(clause_category);
```

### 2.1 设计理由
- **一行一条款**：同一合同可有多条（逾期/验收/保密/解除），天然支持 7 类必含 + 2 类可选。
- **`rate_pct` + `ld_base`**：结构化存「比例 + 基数」，避免只存文本；支持后续「按违约金排序/筛选」。
- **`threshold_days`**：捕获逾期解除阈值（10/60 不等）。
- **`refund_full`**：标记该违约情形是否「退还全部款项」。
- **`clause_text`**：保留原文片段，满足回溯与人工复核。

### 2.2 `clause_category` 取值与覆盖率（来自 #3.①）
| category | 覆盖率 | 是否必填 |
|----------|--------|----------|
| breach_liability | 100% | 必 |
| compensation | 100% | 必 |
| confidentiality | 100% | 必 |
| ip | 100% | 必 |
| liquidated_damages | 100% | 必 |
| overdue | 100% | 必 |
| termination | 100% | 必 |
| force_majeure | 49% | 可选 |
| penalty | 26% | 可选 |

## 3. `contracts` 表变更

```sql
ALTER TABLE contracts ADD COLUMN tax_rate REAL DEFAULT NULL;          -- 税率数值（%）
-- sgsc_id 已存在但为空，回填时直接 UPDATE（不新增列）
```

> 说明：`tax_rate` 存数值（如 6.0）。`sgsc_id` 复用现有列，回填 SGSC 合同编号；`sign_date` 回填时做 `YYYY.M.D`→`YYYY-MM-DD` 归一化。

---

## 4. 字段映射（DOCX 源 → DB）

| DOCX 抽取项 | 目标表.字段 | 处理方式 |
|-------------|------------|----------|
| 合同编号 SGSC... | `contracts.sgsc_id` | 经 `contract_id_mapping` 匹配后 UPDATE |
| 委托方(甲方) | `contracts.party_a` | 已多数存在，OCR 噪声清洗 |
| 受托方(乙方) | `contracts.party_b` | 恒为「北京振华永创智能科技有限公司」 |
| 签订时间 | `contracts.sign_date` | 归一化 `YYYY-MM-DD` |
| 税率 6% | `contracts.tax_rate` | 数值 6.0 |
| 违约责任 9.x 各段 | `contract_clauses` | 按 category/trigger 拆行，rate_pct 取 % |
| 违约金比例 | `contract_clauses.rate_pct` | 占报酬% 数值 |
| 逾期解除阈值 | `contract_clauses.threshold_days` | N 日 |
| 退还全部款项 | `contract_clauses.refund_full` | 命中「退还全部」置 1 |

### 4.1 匹配策略（DOCX → DB 行）
DOCX 文件名含 `ZH02-XXXX` 或 `SGSC...` 片段 → 优先用 `contract_id_mapping` 解析出 `project_id_zh`/`project_id_sgsc` → 匹配 `contracts.contract_id`（该表混存 ZH02 与 SGSC 两种 id，需双键 OR 匹配）→ 定位后回填。无法匹配的合同跳过并记入日志。

## 5. 回填计划（#3.③ 实施）
1. 解析 35 DOCX → 抽取字段 + 条款（复用 `.devtest/_contract_clause_extract.py`）。
2. 按 §4.1 匹配 `contracts` 行。
3. 批量 `UPDATE contracts SET tax_rate=?, sgsc_id=?, sign_date=?` + `INSERT contract_clauses`。
4. OCR 噪声清洗：合同编号小写→大写；日期 `YYYY.M.D`→`YYYY-MM-DD`。
5. 输出回填报告（成功/跳过/异常）。
