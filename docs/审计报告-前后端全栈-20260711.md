## 三、P0 阻断级问题

### P0-01 综合看板 API 返回 500

- **复现**：`GET http://localhost:18080/api/dashboard/overview` → `500 Internal Server Error`。
- **代码位置**：`backend/routers/dashboard.py:133-150`。
- **直接根因**：`invoices` 表至少两行文本字段以 GBK 原始字节写入 SQLite TEXT；Python `sqlite3` 按 UTF-8 解码失败。
- **已定位数据**：`invoices.rowid=72/73` 的 `status`、`invoice_type`、`payment_status`、`notes` 存在非法 UTF-8。
- **影响**：综合看板不可用；同类聚合接口可能被任意一条脏文本拖垮。
- **分类**：数据质量为直接根因，导入链路缺少编码规范化是代码缺陷。
- **建议**：先修复两行数据，再在财务导入入口拒绝或转换 `bytes`；增加脏编码回归测试。禁止用 `text_factory=bytes` 掩盖问题。

### P0-02 两个合同详情 API 返回 500

稳定版 42 合同巡检发现：

| 合同 ID | 直接错误 | 代码位置 |
|---|---|---|
| `SGSCLSOOSDJS250072` | 发票 `invoice_type` 非 UTF-8，查询解码失败 | `backend/routers/contracts.py:186-192` |
| `ZH02-202604023` | `start_time` 为 `NULL`，直接传给 `re.search()` | `backend/routers/contracts.py:209-213` |

- **影响**：合同详情页只能显示错误态，KPI、阶段、付款、发票、交付物、条款均不可用。
- **最小修复**：
  - 清理非法编码发票记录并修复导入链路。
  - 将 `start_val = s.get('start_time') or ''`，并增加 `NULL` 阶段时间测试。

### P0-03 供应商详情页面不可用

- **运行态**：`/web/suppliers/detail?id=SUP-64997` 显示“供应商详情加载失败 / 内部服务器错误”。
- **API 实测**：`GET /api/suppliers/SUP-64997` 实际返回 200。
- **根因**：详情接口返回裸对象 `{supplier, contracts}`，而全局请求拦截器要求 `{code:0,data:...}`。
- **代码位置**：
  - `backend/routers/suppliers.py:22-40`
  - `ui-vben/apps/web-antd/src/api/request.ts:74-81`
  - `ui-vben/apps/web-antd/src/views/suppliers/detail.vue:78-91`
- **最小修复**：后端详情统一 `return vben_response({...})`；补前后端契约测试。

## 四、P1 高风险问题

### P1-01 发票详情金额单位错误 10,000 倍

- **API 示例**：发票 313 的 `amount=31076.5`，数据库口径为“元”。
- **页面实际显示**：`31076.50 万元`。
- **正确显示应约为**：`3.10765 万元`。
- **代码位置**：`ui-vben/apps/web-antd/src/views/invoices/detail.vue:14`。
- **影响**：财务金额严重失真。
- **建议**：显示时 `/10000`，或由后端 DTO 明确返回 `amount_yuan` 与 `amount_wan`，禁止含糊字段名。

### P1-02 项目列表存在确定的 JavaScript 运行时错误

- **运行态错误**：`TypeError: y.on is not a function`。
- **源码**：`ui-vben/apps/web-antd/src/views/projects/index.vue:173-190` 使用不存在的 `gridApi.on()` 与 `commitProxyQuery()`。
- **类型检查同时报错**：TS2339。
- **违反 Vben 规则**：项目已有正确模式应通过 `gridEvents` 和框架支持的 API。
- **影响**：双击跳转、搜索、重置等交互可能失效。

### P1-03 认证授权为开发 mock，业务接口无有效保护

- **代码位置**：`backend/routers/auth.py:10-71`。
- 登录不校验账号密码，固定返回开发 token；权限固定为管理员和全权限。
- 业务路由未见统一鉴权依赖或中间件。
- **影响**：任何能够访问后端的人都可能读取或修改合同、发票、供应商数据。
- **建议**：开发 mock 必须按环境隔离；生产启用真实 token 校验和路由级权限。

### P1-04 CORS 全开放

- **代码位置**：`backend/main.py:20`。
- `allow_origins=['*']`、方法与请求头全开放。
- **影响**：与无鉴权 API 叠加后，任意网页可直接跨域调用业务接口。
- **建议**：按环境配置明确白名单。

### P1-05 合同 ID 映射非唯一且查询结果不确定

- `contract_id_mapping.project_id_sgsc` 没有唯一约束。
- 当前数据库存在 **16 组**同一 SGSC ID 映射到多个 ZH/SYS ID。
- `backend/routers/contracts.py:125-132` 使用无 `ORDER BY` 的 `fetchone()` 选择映射。
- 发票、项目、阶段、条款等接口没有统一解析全部别名。
- **影响**：漏数、串单、重复统计或不同环境返回不同关联数据。
- **建议**：建立 canonical ID + alias 表；禁止在冲突未清理前直接加唯一索引。

### P1-06 付款计划存在极端单位和重复导入异常

只读数据库核验发现：

- `ZH02-202512056` 合同金额 `17.0766 万元`，付款计划合计 `5156.6409 万元`。
- 其中一条付款 `planned_amount=5123.0`，明显可能把“0.5123 万元”写成了“5123 万元”。
- 同一业务付款同时存在规范 ID 与 SGSC 导入副本。
- `ZH02-202604026` 当前合同金额 `82.8 万元`，付款计划合计 `108.438 万元`；其 `actual_amount` 又使用“元”口径。

**影响**：付款进度、回款率和合同余额不可可信。

### P1-07 源码、动态菜单和已构建产物存在路由漂移

- 源码静态路由使用 `detail/:id`，动态菜单使用 `detail` + query 参数。
- 列表 `CellRouterLink` 实际传 query，因此 `/detail?id=...` 可用，而 `/detail/:id` 在当前动态路由中为 404。
- `src/router/routes/modules/dashboard.ts:9` 引用不存在的 `views/dashboard/overview/index.vue`。
- 当前运行页面依赖历史构建产物，源码无法证明可重复构建出同一版本。
- **影响**：本地源码、权限菜单和部署产物行为不一致，修复后容易再次回归。

### P2-07 项目数据完整性不足

- 项目总数：50。

- **影响**：项目详情页团队信息和时间计划无法满足审计清单。

### P2-08 数据库连接与错误路径治理不足

- `backend/database.py:9-12` 每次直接创建 SQLite 连接，无统一上下文、超时、WAL/readiness 策略。
- 多个 404 分支抛异常前未关闭连接，例如 `projects.py:73-76`、`suppliers.py:24-27`。
- **影响**：并发写入或错误请求较多时可能产生锁库与资源泄漏。
