# 立项文档：「回头优化」UI 视觉精修

> 立项日期：2026-07-09 ｜ 负责人：pm-director ｜ 状态：待评审
> 设计准则基线：ardot-design-core（配色单一来源 / 状态可见性 / 一致性 / 密度）、ardot-design-router（导航与路由一致性）
> 代码基线：`docs/audit-ui-layout-20260708.md`、`ui-vben/apps/web-antd/src/styles/finance-tokens.css`

---

## 一、项目背景与目标

P2/P3 阶段已交付仪表盘 + 合同/发票/供应商详情页，功能可用，但经审计发现**视觉语言不统一、状态反馈缺失、存在误导性展示**。本轮「回头优化」目标是把已落地的一致性地基（finance-tokens.css、StateBlock 组件）转化为**全站统一的视觉规范与体验**，让看板与详情页不再是「两套语言」。

**目标（可量化）**
- 全站财务/状态语义色 100% 走 `finance-tokens.css` 单一来源，0 处业务组件硬编码 hex。
- 3 个详情页 100% 具备 loading/error/empty 三态，消灭「接口失败即白屏」。
- 看板与详情页收敛为**同一套卡片体系**与间距/圆角/标题层级 token。
- 阶段进度消除「假百分比」误导。

## 四、设计原则（ardot）

1. **配色单一来源**：所有语义色集中在 `finance-tokens.css`，组件禁止散落 hex。
2. **状态可见性（System Status）**：每个数据视图必须有 loading / error（含重试）/ empty 三态，失败可见、为空可懂。
3. **一致性（Consistency）**：同一概念（卡片、按钮、标签、表格）在全站同一视觉语言；导航/路由跳转方式统一。
4. **密度（Density）**：基于 8pt 栅格统一间距与圆角，避免「这里紧那里松」。

## 五、需求池

### P0（必须，影响信任与可用性）
- **P0-1** 三态补齐：合同/发票/供应商详情页接入 `components/state-block/index.vue`（loading/error/empty），消灭白屏。
- **P0-2** 全站语义色统一：图表组件、详情页硬编码 hex 全部替换为 `finance-tokens.css` 变量；状态色（`--status-*`）与甘特图 JS 色板对齐。

### P1
- **P1-1** 删除死代码 `views/dashboard/index.vue`。
- **P1-2** 统一卡片体系：收敛看板 `border+bg-card` 与详情 `a-card` 为一套 `.panel` token（border 1px / radius 12 / shadow-sm / bg-card）。
- **P1-3** 阶段进度修正：改为真实完成度，或改标签为「状态指示」，去除误导。
- **P1-4** 统一间距/圆角/标题层级 token（8pt 栅格；page-title 20/600、section-title 16/600、card-title 14/600；卡片 radius 12、标签/按钮 6/4）。

### P2（体验增益，部分依赖 #3）
- **P2-1** 合并 `finance-trend-chart` 与 `monthly-trend-chart` 为参数化组件。
- **P2-2** 落地 ECharts 甘特图（**依赖 #3 stage 数据**）。
- **P2-3** 合同文件上传（后端端点 + 前端组件，**依赖 #3 文件存储**）。
- **P2-4** 供应商详情关联合同表加跳转链接（打通供应商↔合同）。

## 八、风险与待确认

1. **甘特图/上传依赖 #3**：M3 必须等数据补全的 stage 表与文件存储方案落地。
2. **卡片统一方向待确认**：保留 `a-card` 还是全改 `border+bg-card`？建议全站统一为 `.panel`（border+bg-card 风格），需你拍板。
3. **视觉回归基线**：替换硬编码色后需用 Playwright 截图做像素级回归，避免「修色即变色」。
4. **图表颜色来源**：`finance-trend-chart` 等组件的颜色应从 token 注入，而非写死。

## 九、验收标准

- `grep -r` 业务组件无硬编码财务 hex（仅 `finance-tokens.css` 含色值）。
- 3 个详情页在「无 id / 接口 500 / 空数据」三种场景下均显示正确三态，0 白屏。
- 看板与详情页卡片视觉一致（对照设计 token 截图评审通过）。
- Playwright 全量回归 0 FAIL / 0 控制台错误。
