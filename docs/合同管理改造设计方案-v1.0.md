## 一、改造背景

### 1.1 现状问题

随着项目数量增加，发现以下系统性问题：

| 问题类型 | 具体表现 | 影响 |
|---------|---------|------|
| **数据孤岛** | 财务表格、飞书表格、合同文件各自独立 | 数据核对成本高 |
| **字段缺失** | 缺少交付时间、付款条件、交付阶段等关键字段 | 项目进度跟踪不精细 |
| **流程断层** | 合同签订 → 项目执行 → 交付验收 → 开票回款各环节数据未串联 | 风险预警不及时 |
| **识别能力不足** | 合同关键信息提取依赖人工 | 效率低、易出错 |

### 1.2 典型案例

**蜀能项目数据错误**：
- 蜀能高新（ZH02-202601011）和蜀能电科（ZH02-202601010）的项目编号在财务表格和飞书表格中**完全相反**
- 导致开票回款数据交叉错误
- 2026-06-28发现并修正

**HPLC项目金额不准**：
- 涉及到的所有金额数据都不准确
- 需要等最后一批到货确认后才能统计
- 临时方案：不显示金额，只显示发货进度

## 二、改造目标

### 2.1 核心思想

**"合同管理的改造同时提升项目管理的精细化程度和规范程度"**

### 2.2 目标分解

#### 1. 精细化管理
- 从"项目整体" → "项目阶段"
- 从"整体进度" → "阶段进度"
- 从"整体金额" → "阶段付款"
- 从"模糊状态" → "精确节点"

#### 2. 规范化管理
- 数据字段标准化
- 统计口径标准化
- 编号体系标准化
- 流程节点标准化

#### 3. 系统性改造
- 不是单点修补，而是体系重构
- 不是临时方案，而是长期机制
- 不是工具优化，而是能力升级

## 三、字段体系设计

### 3.1 新增字段（10个）

#### 【精细化进度】（4个）
- **交付阶段**：合同约定的阶段划分
- **当前阶段**：项目当前所处阶段
- **阶段进度**：当前阶段完成百分比
- **交付时间**：合同约定的交付截止时间

#### 【付款计划】（2个）
- **付款条件**：与交付阶段对应的付款计划
- **阶段付款状态**：每个阶段的付款情况

#### 【合同信息】（2个）
- **合同签订日期**：合同签订的时间
- **合同编号**：合同编号（与项目编号区分）

#### 【风险管理】（2个）
- **风险等级**：项目整体风险评级
- **风险描述**：具体风险说明

### 3.2 字段关联关系

```
合同编号 ←→ 项目编号（一一对应）

交付阶段 ←→ 付款条件（阶段与付款对应）
    ↓
当前阶段 ←→ 阶段进度（进度跟踪）
    ↓
阶段付款状态（付款跟踪）

风险等级 ←→ 风险描述（风险管理）
```

## 四、流程节点标准化

### 4.1 标准流程（7个节点）

```
【节点1：合同签订】
触发：合同签订完成
操作：飞书表格新增记录 + 合同信息字段自动填充

【节点2：项目启动】
触发：项目启动会召开
操作：更新项目经理、分包单位字段 + 启动会沟通记录

【节点3：阶段执行】（循环）
触发：项目进入某个阶段
操作：更新阶段进度 + 沟通记录增量追加

【节点4：阶段交付】（循环）
触发：某个阶段交付完成
操作：更新实际交付时间 + 交付物记录

【节点5：阶段开票】（循环）
触发：某个阶段开票完成
操作：更新已开票金额 + 开票记录

【节点6：阶段回款】（循环）
触发：某个阶段回款到账
操作：更新已回款金额 + 回款记录

【节点7：项目完成】
触发：项目最终验收
操作：更新项目状态为"已完成" + 项目总结记录
```

## 五、合同识别能力增强

### 5.1 需要增强的能力

#### 1. 交付阶段识别
- 关键词识别："第一阶段"、"阶段一"、"一期"等
- 表格解析：识别包含"阶段"、"内容"、"时间"的表格
- 语义理解：理解"方案设计"、"系统开发"等阶段描述

#### 2. 付款条件匹配
- 关键词匹配："第一阶段付款"、"首付款"等
- 金额提取：识别每个阶段的付款金额或比例
- 对应关系推断：推断阶段与付款的对应关系

#### 3. 交付内容提取
- 关键词识别："交付物"、"成果"、"输出"等
- 表格解析：识别包含"阶段"、"交付物"的表格
- 语义理解：理解具体交付物描述

## 六、数据同步机制

### 6.1 同步策略

**以财务表格为唯一数据源**

### 6.2 每周同步流程

```
财务表格发送 → 对比飞书表格 → 发现差异 → 更新飞书表格 → 验证同步结果
```

## 八、预期成果

### 8.1 系统能力提升

| 能力维度 | 改造前 | 改造后 |
|---------|--------|--------|
| 数据准确性 | 财务表格与飞书表格不一致 | 以财务表格为准，自动同步 |

| 数据核对 | 每周2小时 | 每周30分钟 | **4倍** |
