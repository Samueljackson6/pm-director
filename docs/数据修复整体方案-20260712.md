## 三、问题3修复执行计划

### 阶段1: 批量分类（0.5天）

对23个无付款服务类合同的OCR进行付款模式分类：
- 类型A: 一次性付款（约3-5个）
- 类型B: 按比例分期（约10-15个）
- 类型C: 需人工确认（约5-8个）

### 阶段2: 自动修复（0.5天）

对类型A和类型B合同，批量生成付款记录：

```sql
-- 示例：ZH02-202508013（一次性付款）
INSERT INTO payments (payment_id, contract_id, planned_amount, payment_stage, status)
VALUES (
  'PAY-ZH02-202508013-1',
  'ZH02-202508013',
  80.8356,
  '技术服务内容全部完成，经甲方验收合格，并收到乙方提供的合格增值税发票一个月内支付合同金额100%',
  'pending'
);

-- 示例：ZH02-202406004（80%+20%分期）
INSERT INTO payments (payment_id, contract_id, planned_amount, payment_stage, status)
VALUES 
('PAY-ZH02-202406004-1', 'ZH02-202406004', 45.78, '验收合格后支付80%进度款', 'pending'),
('PAY-ZH02-202406004-2', 'ZH02-202406004', 11.45, '交齐竣工资料并归档后支付剩余款项', 'pending');
```

### 阶段3: 人工确认（1天）

对类型C合同：
1. 列出待确认清单
2. 人工阅读OCR原文
3. 确定付款节点和金额
4. 手动录入数据库

### 阶段4: 验证（0.5天）

1. 重新运行付款差异统计
2. 确认完全匹配比例达到80%+
3. 前端展示验证
