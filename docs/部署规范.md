> 更新: 2026-07-02 | 体系版本: v2.0 (自动化)

---

## 一、环境总览

| 环境 | 地址 | 用途 |
|------|------|------|
| **236 生产** | `192.168.0.236` (samuel) | FastAPI 后端 (8800) + Vben 前端 (5777) |
| **GitHub** | `github.com/Samueljackson6/pm-director` | 代码托管 + CI 验证 |

## 二、自动化体系架构

```
开发者本地
  │
  ├── 1. 编辑代码 (backend/ 或 ui/)
  ├── 2. 运行 lint 检查 (scripts/auto-publish.sh)
  ├── 3. git add + commit + push
  │
  ├── GitHub Actions CI (自动执行)
  │   ├── Python lint (ruff + 语法检查)
  │   ├── UI 文件完整性检查
  │   └── 提交信息格式检查
  │
  └── 4. 一键部署到 236
      ├── bash scripts/auto-publish.sh deploy
      ├── 或 ssh 236 "bash scripts/deploy-236.sh all"
      └── 自动验证 + 审计
```

### 自动化体系状态

| 组件 | 状态 | 说明 |
|------|------|------|
| GitHub Actions CI | ✅ 已验证通过 | Python Lint + UI Validate + 自动部署 |
| 236 自托管 Runner | ✅ 已配置运行 | 监听 GitHub 任务，自动执行部署 |
| systemd --user 服务 | ✅ 已安装启用 | 后端/前端自动崩溃重启，无 sudo 需求 |
| 一键部署 | ✅ 已验证 | `bash scripts/deploy-236.sh all` 或 CI 自动触发 |

### 关键约束

- 236 服务器无公网 IP → GitHub Actions 使用 **自托管 Runner** 部署（Runner 在 236 本地运行）
- 禁止 `rsync -a --delete` 到生产目录 → 会删除 `database/` `cache/` `ui-vben/` 等运行时数据
- 服务管理必须使用 **systemd --user**（禁用 nohup/pkill 方式）

## 三、后端部署

### 3.1 systemd --user 服务管理（强制）

```bash
# 重启后端
systemctl --user restart pm-backend

# 查看状态
systemctl --user status pm-backend

# 查看日志
journalctl --user -u pm-backend -n 50 -f

# 统一管理脚本
bash /home/samuel/.openclaw/workspace/pm-director/scripts/start-services.sh restart
```

### 3.2 手动命令（备用）

```bash
ssh 236
cd /home/samuel/.openclaw/workspace/pm-director

# 拉取最新代码
git pull origin master

# 重启
pkill -f 'uvicorn backend.main' 2>/dev/null
nohup python3 -m uvicorn backend.main:app --host 0.0.0.0 --port 8800 > /tmp/pm-backend.log 2>&1 &

# 验证
curl http://localhost:8800/api/stats
```

## 四、前端部署

### 4.1 完整部署流程

```bash
ssh 236
cd /home/samuel/.openclaw/workspace/pm-director

# Step 1: 拉取最新代码
git pull origin master

# Step 2: 同步自定义 UI 代码到 Vben 框架
cp ui/src/api/*.ts ui-vben/apps/web-antd/src/api/
cp ui/src/views/contracts/index.vue ui-vben/apps/web-antd/src/views/contracts/
cp ui/src/views/contracts/detail.vue ui-vben/apps/web-antd/src/views/contracts/
cp ui/src/views/invoices/index.vue ui-vben/apps/web-antd/src/views/invoices/
cp ui/src/views/suppliers/index.vue ui-vben/apps/web-antd/src/views/suppliers/
cp ui/src/router/routes/modules/*.ts ui-vben/apps/web-antd/src/router/routes/modules/

# Step 3: 构建
cd ui-vben && npx pnpm -F @vben/web-antd build

# Step 4: 重启前端服务
sudo systemctl restart pm-frontend

# Step 5: 验证
curl http://localhost:5777/
curl http://localhost:8800/api/stats
```

### 4.2 systemd 服务管理（推荐）

```bash
# 重启前端
sudo systemctl restart pm-frontend

# 查看状态
sudo systemctl status pm-frontend

# 查看日志
sudo journalctl -u pm-frontend -n 50 -f
```

## 五、一键部署

### 5.1 一键发布（完整流水线）

```bash
# 从本地执行（需要能 SSH 到 236）
bash scripts/auto-publish.sh

# 流程：
# 1. 检查前置条件
# 2. Lint 检查
# 3. Git 提交 + 推送
# 4. 236 部署（git pull → copy → build → restart）
# 5. 验证（Frontend HTTP 200, Backend HTTP 200）
# 6. 审计（完成度评估）
```

### 5.2 仅部署

```bash
# 代码已推送时，仅执行部署
bash scripts/auto-publish.sh deploy

# 或在 236 上直接执行
ssh 236 "cd /home/samuel/.openclaw/workspace/pm-director && bash scripts/deploy-236.sh all"
```

### 5.3 部署子命令

```bash
bash scripts/deploy-236.sh all       # 后端+前端部署
bash scripts/deploy-236.sh backend   # 仅后端
bash scripts/deploy-236.sh frontend  # 仅前端
bash scripts/deploy-236.sh verify    # 仅验证
```

## 六、CI/CD 管道

### 6.1 GitHub Actions

文件: `.github/workflows/ci.yml`

| 触发 | 操作 |
|------|------|
| Push to master | Python lint + UI 验证 + 提交信息检查 |
| Push to any branch | Python lint + UI 验证 |
| Pull Request | Python lint + UI 验证 |

### 6.2 CI 检查项

1. **Python Lint** — ruff 检查 + 语法验证

3. **路由规范** — `path: ''` 格式检查
4. **提交信息格式** — `type(scope): message` 验证

## 七、服务管理

### 7.1 systemd 服务

| 服务 | 配置位置 | 端口 | 管理命令 |
|------|---------|------|---------|
| 后端 pm-backend | `systemd/pm-backend.service` | 8800 | `sudo systemctl restart pm-backend` |
| 前端 pm-frontend | `systemd/pm-frontend.service` | 5777 | `sudo systemctl restart pm-frontend` |

### 7.2 安装 systemd 服务

```bash
# 在 236 上执行一次
sudo cp /home/samuel/.openclaw/workspace/pm-director/systemd/pm-backend.service /etc/systemd/system/
sudo cp /home/samuel/.openclaw/workspace/pm-director/systemd/pm-frontend.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now pm-backend
sudo systemctl enable --now pm-frontend
```

### 7.3 日常维护

```bash
# 查看服务状态
sudo systemctl status pm-backend pm-frontend

# 重启
sudo systemctl restart pm-backend pm-frontend

# 查看日志
sudo journalctl -u pm-backend -n 30 --no-pager
sudo journalctl -u pm-frontend -n 30 --no-pager

# 跟踪日志
sudo journalctl -u pm-backend -f
```

## 八、故障处理

### 8.1 前端页面空白

| 可能原因 | 检查方式 | 修复 |
|---------|---------|------|
| 路由错误 | `grep 'path:' ui/src/router/routes/modules/*.ts` | 子路由用 `path: ''` |
| 构建错误 | `sudo journalctl -u pm-frontend -n 50` | 重新构建 |
| 服务未启动 | `sudo systemctl status pm-frontend` | `sudo systemctl restart pm-frontend` |
| 浏览器缓存 | 硬刷新 (Ctrl+Shift+R) | 清缓存 |
| API 跨域 | 浏览器 Console 看 CORS 错误 | 检查 backend CORS 配置 |

### 8.2 后端错误

```bash
# 查看日志
sudo journalctl -u pm-backend -n 50 --no-pager

# 重启
sudo systemctl restart pm-backend

# 测试 API
curl http://localhost:8800/api/stats
```

### 8.3 服务全部挂掉

```bash
# 1. 检查所有服务
sudo systemctl status pm-backend pm-frontend

# 2. 逐一重启
sudo systemctl restart pm-backend
sudo systemctl restart pm-frontend

# 3. 验证
curl http://localhost:5777/
curl http://localhost:8800/api/stats

# 4. 如果 systemd 失败，手动启动备用
cd /home/samuel/.openclaw/workspace/pm-director
nohup python3 -m uvicorn backend.main:app --host 0.0.0.0 --port 8800 &
cd ui-vben/apps/web-antd/dist
nohup python3 -m http.server 5777 --bind 0.0.0.0 &
```
