## 零、服务器连接方式

### 前提条件

- SSH 密钥已配置：`~/.ssh/id_ed25519_236`（236服务器专用 ed25519 密钥）
- SSH config 已配置（见 `~/.ssh/config`），定义了 `236` 和 `236-frp` 两个 Host

### 方式一：局域网免密登录（推荐，同局域网下使用）

```bash
# 直接使用 SSH config 别名
ssh 236

# 或全量命令
ssh -i ~/.ssh/id_ed25519_236 samuel@192.168.0.236
```

### 方式二：互联网穿透连接（通过阿里云跳板，异地使用）

236 服务器无公网 IP，通过阿里云服务器（47.98.184.188）配置 SSH 远程端口转发暴露 `58022` 端口到公网：

```
你的电脑 ──SSH:58022──→ 阿里云 47.98.184.188 ──隧道──→ 236 服务器 192.168.0.236:22
```

```bash
# 使用 SSH config 别名
ssh 236-frp

# 或全量命令
ssh -p 58022 -i ~/.ssh/id_ed25519_236 samuel@47.98.184.188
```

> **阿里云转发原理**：阿里云 sshd 开启了 `GatewayPorts yes`，236 服务器通过 SSH 反向隧道连接到阿里云，将自身的 22 端口映射到阿里云的 58022 端口。

### SSH Config 参考

```text
# ~/.ssh/config
# 局域网直连
Host 236
    HostName 192.168.0.236
    User samuel
    Port 22
    IdentityFile ~/.ssh/id_ed25519_236
    StrictHostKeyChecking no
    ServerAliveInterval 60

# 互联网穿透（通过阿里云）
Host 236-frp
    HostName 47.98.184.188
    User samuel
    Port 58022
    IdentityFile ~/.ssh/id_ed25519_236
    StrictHostKeyChecking no
    ServerAliveInterval 60
```

### 验证连接

```bash
# 检查主机名（应返回 SamuelUbuntu）
ssh 236-frp "hostname"

# 检查服务状态
ssh 236-frp "systemctl --user status pm-backend | head -5"
```

## 一、环境总览

| 环境 | 地址 | 用途 |
|------|------|------|
| **236 生产** | `192.168.0.236` (samuel) | FastAPI 后端 (8800) + Vben 前端 (5777) |
| **GitHub** | `github.com/Samueljackson6/pm-director` | 代码托管 + CI 验证 |

## 二、自动化体系架构

```
开发者本地
  │
  ├── 1. 编辑代码 (backend/ 或 ui/)
  ├── 2. 运行 lint 检查 (scripts/auto-publish.sh)
  ├── 3. git add + commit + push
  │
  ├── GitHub Actions CI (自动执行)
  │   ├── Python lint (ruff + 语法检查)
  │   ├── UI 文件完整性检查
  │   └── 提交信息格式检查
  │
  └── 4. 一键部署到 236
      ├── bash scripts/auto-publish.sh deploy
      ├── 或 ssh 236 "bash scripts/deploy-236.sh all"
      └── 自动验证 + 审计
```

### 自动化体系状态

| 组件 | 状态 | 说明 |
|------|------|------|
| GitHub Actions CI | ✅ 已验证通过 | Python Lint + UI Validate + 自动部署 |
| 236 自托管 Runner | ✅ 已配置运行 | 监听 GitHub 任务，自动执行部署 |
| systemd --user 服务 | ✅ 已安装启用 | 后端/前端自动崩溃重启，无 sudo 需求 |
| 一键部署 | ✅ 已验证 | `bash scripts/deploy-236.sh all` 或 CI 自动触发 |

### 关键约束

- 236 服务器无公网 IP → 通过阿里云端口转发暴露 `58022` 端口（详见「零、服务器连接方式」）。GitHub Actions 使用 **自托管 Runner** 部署（Runner 在 236 本地运行）
- 禁止 `rsync -a --delete` 到生产目录 → 会删除 `database/` `cache/` `ui-vben/` 等运行时数据
- 服务管理必须使用 **systemd --user**（禁用 nohup/pkill 方式）

## 三、后端部署

### 3.1 systemd --user 服务管理（强制）

```bash
# 重启后端
systemctl --user restart pm-backend

# 查看状态
systemctl --user status pm-backend

# 查看日志
journalctl --user -u pm-backend -n 50 -f

# 统一管理脚本
bash /home/samuel/.openclaw/workspace/pm-director/scripts/start-services.sh restart
```

### 3.2 手动命令（备用）

```bash
ssh 236
cd /home/samuel/.openclaw/workspace/pm-director

# 拉取最新代码
git pull origin master

# 重启
pkill -f 'uvicorn backend.main' 2>/dev/null
nohup python3 -m uvicorn backend.main:app --host 0.0.0.0 --port 8800 > /tmp/pm-backend.log 2>&1 &

# 验证
curl http://localhost:8800/api/stats
```

## 七、服务管理

### 7.1 systemd 服务

| 服务 | 配置位置 | 端口 | 管理命令 |
|------|---------|------|---------|
| 后端 pm-backend | `systemd/pm-backend.user.service` | 8800 | `systemctl --user restart pm-backend` |
| 前端 pm-frontend | `systemd/pm-frontend.user.service` | 5777 | `systemctl --user restart pm-frontend` |

### 7.2 安装 systemd --user 服务（无需 sudo）

```bash
# 在 236 上执行一次
mkdir -p ~/.config/systemd/user/
cp /home/samuel/.openclaw/workspace/pm-director/systemd/pm-backend.user.service ~/.config/systemd/user/pm-backend.service
cp /home/samuel/.openclaw/workspace/pm-director/systemd/pm-frontend.user.service ~/.config/systemd/user/pm-frontend.service
systemctl --user daemon-reload
systemctl --user enable --now pm-backend
systemctl --user enable --now pm-frontend
```

### 7.3 日常维护

```bash
# 查看服务状态
systemctl --user status pm-backend pm-frontend

# 重启
systemctl --user restart pm-backend pm-frontend

# 查看日志
journalctl --user -u pm-backend -n 30 --no-pager
journalctl --user -u pm-frontend -n 30 --no-pager

# 跟踪日志
journalctl --user -u pm-backend -f
```

## 八、故障处理

### 8.1 前端页面空白

| 可能原因 | 检查方式 | 修复 |
|---------|---------|------|
| 路由错误 | `grep 'path:' ui/src/router/routes/modules/*.ts` | 子路由用 `path: ''` |
| 构建错误 | `journalctl --user -u pm-frontend -n 50` | 重新构建 |
| 服务未启动 | `systemctl --user status pm-frontend` | `systemctl --user restart pm-frontend` |
| 浏览器缓存 | 硬刷新 (Ctrl+Shift+R) | 清缓存 |
| API 跨域 | 浏览器 Console 看 CORS 错误 | 检查 backend CORS 配置 |

### 8.2 后端错误

```bash
# 查看日志
journalctl --user -u pm-backend -n 50 --no-pager

# 重启
systemctl --user restart pm-backend

# 测试 API
curl http://localhost:8800/api/stats
```

### 8.3 服务全部挂掉

```bash
# 1. 检查所有服务
systemctl --user status pm-backend pm-frontend

# 2. 逐一重启
systemctl --user restart pm-backend
systemctl --user restart pm-frontend

# 3. 验证
curl http://localhost:5777/
curl http://localhost:8800/api/stats

# 4. 如果 systemd --user 失败（session 重启后），先登录用户 session：
loginctl enable-linger samuel
# 然后手动启动：
cd /home/samuel/.openclaw/workspace/pm-director
nohup python3 -m uvicorn backend.main:app --host 0.0.0.0 --port 8800 &
cd /home/samuel/ruoyi-office-vben/apps/web-antd/dist
nohup python3 -m http.server 5777 --bind 0.0.0.0 &
```
