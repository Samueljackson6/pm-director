## 二、CI/CD（GitHub Actions）

文件：`.github/workflows/ci.yml`。单 `master` 分支，`paths-ignore: docs/**, *.md`。

| Job | Runner | 内容 |
|-----|--------|------|
| python-lint | GitHub-hosted | `ruff` + `ast.parse` 语法校验 |
| ui-validate | GitHub-hosted | `ui/src/...` 关键文件存在性 + `path: ''` 路由格式 |
| deploy | **自托管 Runner (236)** | rsync → `deploy-docker.sh` → 验证 → 审计 |

> ⚠️ **CI 现状**：仅做静态检查 + Docker 部署，**不运行任何单测**（仓库暂无 pytest/vitest）。
> ⚠️ **CI 验证端口 BUG**：`ci.yml` 与 `deploy-docker.sh` 的验证步骤 `curl http://localhost:5777/` 用的是老环境端口；纯 Docker 下前端在 `8900`/`/web/`，需一并修正为 `http://localhost:8900/web/`。

### 本地一键

```bash
bash scripts/auto-publish.sh            # lint + commit + push
bash scripts/auto-publish.sh deploy     # 仅触发部署
```

## 三、部署方式 A：【目标】全 Docker（推荐，CI 采用）

### 3.1 一键部署（CI 自动 / 手动）

```bash
# 在 236 上（CI 自托管 Runner 已自动执行此路径）
cd /home/samuel/.openclaw/workspace/pm-director
bash scripts/deploy-docker.sh
#   内部：git pull → docker compose build --no-cache → docker compose down → docker compose up -d → 验证+审计
```

### 3.2 端口与访问映射

| 服务 | 容器 | 宿主映射 | 访问路径 |
|------|------|---------|---------|
| 后端 | `pm-director-backend` :8800 | `:8800` | `http://<host>:8800/api/stats` |
| 前端 | nginx :80 | `:8900` | `http://<host>:8900/web/`（根路径 301 到 /web/） |

健康检查：`http://localhost:8900/health` → 200；后端 `http://localhost:8800/api/stats`。

### 3.3 ⚠️ 前端 dist 构建须知（当前坑）

`docker-compose.yml` 的 `frontend` 服务使用 `image: nginx:alpine` + 挂载宿主 `./ui-vben/apps/web-antd/dist`，**不会自动从 `Dockerfile.frontend` 构建**。因此：

- `docker compose build` 实际只重建了 backend；
- `deploy-docker.sh` 的 rsync 同步的是源码，**不会自动产出前端 dist**；
- **务必在部署前先构建前端产物**，二选一：
  ```bash
  # 方案 1（快，宿主构建）：在 236 上
  cd ui-vben && pnpm install && cd apps/web-antd && pnpm build
  #   → 产物到 ui-vben/apps/web-antd/dist，被 compose 卷挂载

  # 方案 2（全镜像，含 jiti 补丁）：改 docker-compose.yml 让 frontend 用 Dockerfile.frontend
  #   frontend:
  #     build:
  #       context: .
  #       dockerfile: Dockerfile.frontend
  #     image: pm-director-frontend
  #   然后 docker compose build 会真正构建前端镜像（已处理 jiti 内联问题）
  ```

## 九、故障处理

| 现象 | 排查 |
|------|------|
| 前端空白（老环境） | 路由格式 `path: ''`；`journalctl --user -u pm-frontend`；硬刷新清缓存 |
| 前端 8900 返回旧页面 | 宿主 `ui-vben/apps/web-antd/dist` 未更新 → 重新 `pnpm build`（见 §3.3） |
| 后端报错 | `journalctl --user -u pm-backend`；`curl /api/stats` |
| 服务全挂 | Docker：`docker compose restart`；老环境：`systemctl --user restart pm-backend pm-frontend` |
