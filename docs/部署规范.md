## 二、后端部署 (FastAPI)

### 手动部署

```bash
ssh 236
cd /home/samuel/.openclaw/workspace/pm-director

# 拉取最新代码
git pull origin master

# 重启后端服务
pkill -f 'uvicorn backend.main' 2>/dev/null
nohup python3 -m uvicorn backend.main:app --host 0.0.0.0 --port 8800 > /tmp/pm-backend.log 2>&1 &

# 验证
curl http://localhost:8800/api/stats
```

### 状态检查

```bash
curl http://192.168.0.236:8800/api/stats
# 预期: {"code":0,"data":{"contract_count":30,...}}
```

## 三、前端部署 (Vben Admin 5)

### 前提条件

- Vben Admin v5 框架已安装在 236: `/home/samuel/.openclaw/workspace/pm-director/ui-vben/`
- pnpm 已安装 (`npm install -g pnpm`)
- 应用目录: `apps/web-antd/`

### 完整部署流程

```bash
# Step 1: 同步自定义 UI 代码
# pm-director/ui/ 下的代码需要复制到 Vben 框架目录
# 自定义 API 文件
scp ui/src/api/contracts.ts 236:<vben>/apps/web-antd/src/api/
scp ui/src/api/invoices.ts  236:<vben>/apps/web-antd/src/api/
scp ui/src/api/suppliers.ts 236:<vben>/apps/web-antd/src/api/

# 自定义页面
scp ui/src/views/contracts/index.vue 236:<vben>/apps/web-antd/src/views/contracts/
scp ui/src/views/contracts/detail.vue 236:<vben>/apps/web-antd/src/views/contracts/
scp ui/src/views/invoices/index.vue  236:<vben>/apps/web-antd/src/views/invoices/
scp ui/src/views/suppliers/index.vue 236:<vben>/apps/web-antd/src/views/suppliers/

# 自定义路由
scp ui/src/router/routes/modules/contracts.ts 236:<vben>/apps/web-antd/src/router/routes/modules/
scp ui/src/router/routes/modules/invoices.ts  236:<vben>/apps/web-antd/src/router/routes/modules/
scp ui/src/router/routes/modules/suppliers.ts 236:<vben>/apps/web-antd/src/router/routes/modules/

# Step 2: 构建
ssh 236 "cd <vben> && npx pnpm -F @vben/web-antd build"

# Step 3: 重启前端服务
ssh 236 "pkill -f 'http.server 5777' 2>/dev/null; \
  nohup python3 -m http.server 5777 --bind 0.0.0.0 \
  --directory <vben>/apps/web-antd/dist > /tmp/vben5777.log 2>&1 & disown"

# Step 4: 验证
curl http://192.168.0.236:5777/
curl http://192.168.0.236:8800/api/stats
```

### 部署验证清单

- [ ] `curl http://localhost:5777/` → HTTP 200
- [ ] `curl http://localhost:8800/api/stats` → HTTP 200
- [ ] JS bundles 包含 contracts/invoices/suppliers
- [ ] 浏览器访问页面无白屏
- [ ] API 数据正常返回

## 四、GitHub CI/CD

### GitHub Actions 部署工作流

见 `.github/workflows/deploy.yml`（需在 GitHub 仓库配置 `SSH_PRIVATE_KEY` 和 `SERVER_HOST` 等 secrets）。

### 当前限制

- 236 服务器无公网 IP，GitHub Actions 无法直接部署到 236
- 当前采用 **本地推送 → SSH 手动部署** 模式
- CI 自动构建推送 Docker 镜像暂未启用（Docker Desktop 状态不稳定）
