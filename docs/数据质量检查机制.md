> **创建时间**：2026-06-24  
> **维护人**：pm-director

---

## 一、质量检查目标

**核心目标**：确保合同数据完整性、准确性、一致性

**关键指标**：
- 数据完整性：≥ 90%
- 数据准确性：≥ 95%
- 数据一致性：100%

---

## 二、检查维度

### 2.1 完整性检查

**必填字段**：
- ✅ 项目名称
- ✅ 项目编号
- ✅ 客户名称
- ✅ 合同金额
- ✅ 签订日期

**完整性评分**：
```
完整性得分 = 完填字段数 / 必填字段数 × 100%
```

---

### 2.2 准确性检查

**格式验证**：
| 字段 | 格式规则 | 示例 |
|------|---------|------|
| 项目编号 | ZH02-YYYYNNNNN | ZH02-202601001 |
| 签订日期 | YYYY-MM-DD | 2026-01-01 |
| 合同金额 | 数字，单位元 | 635000 |

**内容验证**：
- 合同金额 ≥ 0
- 签订日期在合理范围（2020-2030）
- 项目名称不含特殊字符

---

### 2.3 一致性检查

**逻辑验证**：
```python
# 金额一致性
assert 合同金额 >= 不含税金额
assert 税额 == 不含税金额 * 税率
assert 合同金额 ≈ 不含税金额 + 税额

# 时间一致性
assert 签订日期 >= 项目立项时间
assert 预计交付日期 > 签订日期

# 关系一致性
assert 合同信息记录关联到项目管理表
assert 项目编号唯一（无重复）
```

---

## 三、检查流程

### 3.1 自动检查（入库时）

```
合同入库 → 自动检查 → 记录问题 → 标记待修复 → 提醒负责人
```

**检查项目**：
1. 必填字段完整性
2. 格式正确性
3. 逻辑一致性

---

### 3.2 定期检查（每周）

```
每周一09:00 → 全量检查 → 生成报告 → 修复问题 → 更新统计
```

**检查项目**：
1. 所有合同完整性评分
2. 异常数据标记
3. 缺失字段统计

---

### 3.3 专项检查（按需）

**触发场景**：
- 新合同类型出现
- 发现数据问题
- API恢复后批量导入

**检查项目**：
1. 特定合同类型字段完整性
2. 新增字段数据质量
3. API导入数据验证

## 五、质量改进

### 5.1 自动修复

**可自动修复的问题**：
- 税额计算错误 → 自动重新计算
- 日期格式不一致 → 自动标准化
- 字段关联缺失 → 自动关联

---

### 5.2 人工修复

**需人工修复的问题**：
- 合同金额缺失 → 人工补充
- 签订日期不准确 → 人工校对
- 合同编号错误 → 人工纠正

---

### 5.3 批量修复

**批量修复工具**：
```python
def batch_fix_quality_issues(issues_list):
    for issue in issues_list:
```

## 六、质量标准

### 6.1 质量等级

| 等级 | 完整性 | 准确性 | 一致性 | 说明 |
|------|--------|--------|--------|------|
| **优秀** | ≥ 95% | 100% | 100% | 数据高质量，可直接使用 |
| **良好** | ≥ 85% | ≥ 95% | 100% | 数据质量良好，少量缺失 |
| **合格** | ≥ 70% | ≥ 90% | 100% | 数据基本合格，需补充 |
| **不合格** | < 70% | < 90% | < 100% | 数据质量差，需修复 |

---

### 6.2 质量目标

**短期目标**（本周）：
- 完整性 ≥ 85%
- 准确性 ≥ 95%

**中期目标**（本月）：
- 完整性 ≥ 90%
- 准确性 ≥ 98%

**长期目标**（持续）：
- 完整性 ≥ 95%
- 准确性 100%

## 七、监控与报警

### 7.1 质量监控

**监控指标**：
- 完整性平均得分
- 准确性问题数量
- 一致性错误数量

**报警阈值**：
- 完整性 < 70%：🟡 警告
- 准确性 < 90%：🟠 严重
- 一致性错误 > 0：🔴 紧急

---

**维护说明**：
- 每周一生成质量报告
- 每发现质量问题立即标记
- 每月优化质量标准
