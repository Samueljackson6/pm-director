## 三、登录问题根因分析

### 问题现象

访问 `http://192.168.0.236:5777/` 显示登录页面，输入 admin/admin123 后点击登录，页面不跳转，控制台报错:

```
Failed to load resource: the server responded with a status of 501
@admin-api/system/captcha/get
```

### 根因

**前端 API 基础路径与后端路由不匹配。**

| 配置项 | 前端值 | 后端值 | 匹配? |
|--------|--------|--------|-------|
| API 基础路径 | `/admin-api` | `/` (无前缀) | ❌ 不匹配 |
| 登录接口 | `/admin-api/system/auth/login` | `/system/auth/login` | ❌ |
| 验证码接口 | `/admin-api/system/captcha/get` | `/system/captcha/get` | ❌ |
| 用户信息 | `/admin-api/system/user/info` | `/system/user/info` | ❌ |

### 前端配置

```
# /home/samuel/ruoyi-office-vben/apps/web-antd/.env.production
VITE_GLOB_API_URL=/admin-api
VITE_BASE=/
VITE_BASE_URL=/
```

### 后端实现

```python
# /home/samuel/.openclaw/workspace/pm-director/backend/main.py
@app.post('/system/auth/login')
def vben_login():
    return vben_response({'accessToken': 'vben-dev-token', ...})

@app.post('/system/captcha/get')
def vben_captcha():
    ...

@app.get('/system/user/info')
def vben_user_info():
    return vben_response({...})
```

### 解决方案

**方案A (推荐): 修改前端配置**
- 将 `VITE_GLOB_API_URL=/admin-api` 改为 `VITE_GLOB_API_URL=/`
- 重新构建前端
- 优点: 改动最小，与后端保持一致

**方案B: 修改后端路由**
- 在所有后端路由前加 `/admin-api` 前缀
- 例如: `/admin-api/system/auth/login`
- 缺点: 改动较大，需要修改所有路由

**方案C: 添加反向代理**
- 在 236 上配置 nginx 将 `/admin-api/*` 转发到后端
- 优点: 灵活，不影响代码
- 缺点: 增加配置复杂度

## 七、CI/CD 流水线说明

### 触发流程

```
git push master → GitHub Actions →
  1. Python Lint (GitHub-hosted runner)
  2. UI Validate (GitHub-hosted runner)
  3. 部署到236 (236 self-hosted runner)
     ├── rsync 同步代码 (排除 database/, cache/, ui-vben/)
     ├── pnpm build (Vben 构建)
     ├── systemctl --user restart (服务重启)
     ├── Verify (HTTP 200 检查)
     └── Audit (完成度评估)
```

### 注意事项

1. **禁止使用 `rsync --delete`** - 会删除 database/, cache/, ui-vben/ 等非 git 管理的运行时数据
2. **服务使用 systemd --user** - 无需 sudo，CI 清理不会杀死服务
3. **Git 同步使用 fetch + reset** - 避免凭证问题
