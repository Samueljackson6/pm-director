## 三、关键经验总结

### 3.1 数据质量问题类型

#### 问题1: 多阶段合并付款被错误拆分
**现象**: OCR中"5.1-5.2合并付款15.24万"被拆分为5.1单独+5.2单独，导致金额翻倍

**根因**: 数据提取脚本无法识别合并付款描述

**解决**: 人工核对OCR，按实际描述重新创建付款记录

**建议改进**: 提取脚本应识别"X-Y阶段"模式，创建合并付款记录

#### 问题2: 付款金额重复提取
**现象**: 同一阶段金额被多次提取（如两条18.03万）

**根因**: 去重逻辑缺失

**解决**: 删除重复记录

**建议改进**: 基于(合同ID, 阶段, 金额)组合去重

#### 问题3: 合同金额录入错误
**现象**: ZH02-202604018录入82.8万，实际OCR只有6.22万

**根因**: 数据导入时金额单位换算或录入错误

**解决**: 更正合同金额

**建议改进**: 添加金额校验，OCR金额与录入金额偏差>50%应警告

#### 问题4: 服务类合同付款提取失败
**现象**: 23个服务类合同无付款数据

**根因**: 服务类合同付款条款通常是"按进度付款"，没有明确金额表格

**解决**: 按业务规则(97%+3%质保金)批量创建付款

**建议改进**: 
- 识别"一次性付款"→ 创建97%+3%
- 识别"80%进度款+剩余"→ 按比例创建

### 3.2 数据提取模式总结

#### 科研类合同付款表格格式
```
| 阶段 | 金额 | 付款条件 |
| 5.1-5.2 | 15.24万 | 验收后支付 |
```
**注意**: 多阶段合并需要特殊处理

#### 服务类合同付款条款格式
```
(1) 验收合格后支付合同金额100%
(2) 进度款最高80%，验收后支付97%，剩余3%质保金
(3) 支付50%，验收后支付剩余50%
```

**处理规则**:
- "支付100%" → 97% + 3%质保金
- "80%进度款+剩余" → 80% + 20%
- "验收后支付97%，剩余3%" → 按原文
