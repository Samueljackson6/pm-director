## 1. 阶段目标

阶段 3 的目标不是继续逐页美化，而是先建立可执行、可验收的信息架构和交互基线：

1. 只保留一套全局导航。
2. 重建菜单树和规范路由注册表。
3. 定义列表、详情、关联对象和返回来源的主流程。
4. 建立集中式视觉、数据、状态、可访问性和性能规范。
5. 为“合同台账 → 合同详情 → 项目详情 → 返回”纵向切片提供实施输入。

## 4. 设计决策

### 4.1 信息架构

全局菜单收敛为四个任务域：

- 驾驶舱。
- 合同履约。
- 客户应收。
- 供应商履约。

项目执行正式归入“合同履约”。
供应商档案、供应商发票和付款归入同一履约上下文。

### 4.2 导航

- 桌面端目标布局为 sidebar-nav。
- 顶栏只保留工具。
- 关闭全局页面标签栏。
- 面包屑表达位置。
- 详情页隐藏，并通过来源与返回契约连接。

### 4.3 驾驶舱

保留用户已确认的 /dashboard/overview/index 单路由四视图。
不新增工作台路由，不扩大本阶段范围。

### 4.4 视觉

设计方向从通用“蓝白卡片后台”校准为“履约调度台”。
唯一识别性元素是“履约链”，其余页面保持克制、平直和高可读。

## 6. 验证情况

### 6.1 已验证

- 菜单 API 与磁盘菜单结构一致。
- Vben 官方支持 sidebar-nav、关闭标签栏、后端访问控制和页面级局部 Tab。
- 核心色板在白底和工作台浅灰底上的文字对比度均达到 WCAG AA。
- 目标路由树符合“一级绝对路径、子级相对路径、后端菜单唯一真值”的项目规则。
- 三条关键任务流都定义了规范入口、关联对象和返回来源。
- 当前项目 Legacy 重定向、供应商自重定向和驾驶舱双跳转已记录为纵向切片前置清理项，本轮未声称修复。

### 6.2 证据边界

Playwright 登录后实时复测未在本轮完成。
连续三种脚本写法都被当前 PowerShell 管道的中文选择器编码破坏，因此按失败恢复规则停止继续试错。

本轮没有把 2026-07-13 的截图冒充当前运行结果。
设计依据分为：

- 当前代码知识图谱和精确源码。
- 当前 18080 菜单 API。
- 当前 18090 与 18080 HTTP 可用性。
- 2026-07-13 已归档的真实浏览器截图和交互审计。

纵向切片实现前必须解决 Playwright UTF-8 运行入口，并重新执行登录后的菜单和任务流基线。
