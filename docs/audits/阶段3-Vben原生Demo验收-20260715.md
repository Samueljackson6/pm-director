## 2. 业务设计纠偏

本轮不再使用旧静态 Demo 的通用后台模型，改为围绕当前真实业务成熟度设计：

1. 合同档案工作台：同时展示内部编号、国网编号、法律/交易形式、项目执行形态、金额口径和付款条件。
2. 合同详情：以 PDF 原件、OCR DOCX、结构化抽取、人工核验构成“证据轨道”，每个采用值保留来源和核验状态。
3. 双资金链：客户合同—开票—客户回款与供应商合同—收票—供应商付款严格分开，再通过项目/合同关系连接。
4. 数据核验中心：把历史 `inbound` 语义冲突、SGSC 编号缺失、回款未匹配、阶段—付款关联缺失作为任务，而不是伪装成完整闭环。
5. 移动任务：小程序/App 方向聚焦拍照采集、快速核验、现场验收、同步队列；不复制桌面端大表格。

样例数据均为脱敏评审数据。页面顶部明确标注快照日期、来源和“数字不代表实时经营口径”。

## 3. Vben 兼容性实证

| 页面能力 | 实际组件/模式 | 验证结果 |
|---|---|---|
| 页面壳层 | `Page` + 现有 Vben `BasicLayout` | 通过真实登录和路由验证 |
| 屏幕切换 | Ant Design Vue `Segmented` 显式导入 | 四屏切换通过 |
| 合同档案表 | `useVbenVxeGrid` / `VxeGrid` | 三条脱敏合同、固定列和行跳转通过 |
| 合同详情 | Ant Design Vue `Descriptions`、`Steps`、`Tag` | 双编号、双轴分类、证据轨道通过 |
| 数据核验 | `useVbenDrawer` + 任务卡 | Drawer 打开、Esc 关闭通过 |
| 主题与响应式 | Vben 主题令牌 + CSS 断点 | 1440×1000 与 390×844 通过 |

红灯阶段曾发现当前项目并未全局注册 `a-segmented`，导致四屏切换控件没有真实 DOM。修复为显式导入 `Segmented` 后，同一 Playwright 流程由超时失败转为通过，证明兼容性不是按组件名称推测，而是通过运行时验证。
