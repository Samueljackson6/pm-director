## 1. 完成范围

1. 完成合同、项目组合框的可访问名称与稳定 ARIA 状态处理；在 Ant Design Vue 4.2.x 下避免关闭下拉时保留悬空的 `aria-controls`、`aria-owns` 和 `aria-activedescendant`。
2. 为合同和项目 VxeTable 的水平可滚动表头/表体增加键盘焦点、区域名称，避免宽表滚动区无法被键盘用户进入。
3. 修复正式业务区的可读性：回款语义色、驾驶舱动作链接、客户发票统计卡与非激活标签的对比度均达到浏览器 axe 验收要求。
4. `StateBlock` 的加载、空态和错误态分别具有可读屏的 `role="status"` / `role="alert"`，驾驶舱补齐成功空响应、缺字段错误、陈旧快照提示和冲突数据状态。
5. 新增可重复执行的 `scripts/verify_g006_final_qa_browser.py`，覆盖核心业务任务、四视口、正式驾驶舱、弱网/空/错误/缺失/陈旧/冲突状态、键盘焦点、减少动效、网络写请求、数据库哈希与 axe。

## 2. 无障碍范围与边界

axe 运行范围固定为每个受验核心页面唯一的 `.pm-workbench-page` 业务内容根节点，执行 WCAG 2 A / AA 规则。该范围包含本次阶段 3 重构的页面标题、查询区、表格、统计卡、驾驶舱和状态区；不将 Vben 全局导航、标签壳、通知铃铛等跨页面框架 DOM 混入业务页面结论。

这个边界不是忽略规则：脚本会先断言页面中恰好存在一个业务根节点；缺失或重复会使验收失败。此前全页扫描已定位并记录框架壳层的无名图标按钮、嵌套按钮和标签壳对比度问题，它们不由本次业务纵向切片引入，仍应在框架升级或单独的全局可访问性任务中治理。

最终业务范围 axe 结果：合同、项目、客户发票、供应商、驾驶舱均为 **0 个 Critical / Serious**。

## 4. 未关闭的存量技术债

### 4.1 全仓 TypeScript

`pnpm -F @vben/web-antd run typecheck` 仍失败。错误来自既有资产、BPM、WMS、通用组件和插件类型，不应将本次 G006 表述为全仓类型检查通过。最终基线日志位于本地审计资产 `docs/audits/assets/stage3-g006-web-antd-typecheck-final.log`；本次新增的 `a11y-select.vue`、`use-accessible-vxe-grid.ts`、状态组件和驾驶舱改动均不在最终错误列表中。

### 4.2 全仓 Lint

根 `pnpm lint` 会扫描 `.turbo/cache` 与整个 monorepo，之前在超过 6 分钟后仍未完成；本次未以超时命令伪造通过。对新增可访问性组件、可访问网格 Hook 和 `StateBlock` 执行了受控 ESLint，已通过。既有业务页有大量历史格式/规则债务，需单独治理后再将全仓 Lint 作为硬门禁。

### 4.3 构建警告与框架壳层

生产构建通过，但仍输出 Ant Design Vue 动静态导入混用和部分 chunk 大于 2000 kB 的存量警告。全页 axe 同时发现 Vben 壳层图标按钮、嵌套交互和标签壳对比度问题；这些不在阶段 3 业务根节点范围内，已记录为后续框架层可访问性治理项。

## 5. 评审结论

- **业务真相 / 只读边界**：浏览器加载和下钻未出现业务 API 写请求；数据库哈希在验证前后相同。
- **Vben 边界**：未修改 `src/router/routes/core/`；列表仍经 `#/adapter/vxe-table` 使用 VxeTable；驾驶舱继续使用 Vben `Page`、`StateBlock` 和既有 `requestClient`。
- **视觉与可访问性**：四视口、键盘焦点、减少动效、业务范围 WCAG 2 A/AA axe 均有真实浏览器证据。
- **回归**：后端、全量 Vitest、生产构建和最终浏览器 QA 已通过。
- **独立复核**：尝试启动独立 reviewer，但审查服务于 2026-07-19 返回 `503 Service Unavailable`，未得到可用的独立结论；本报告不把该失败写成“已获无条件批准”。

## 6. 后续建议

1. 新建框架层可访问性任务，处理全局壳层的无名图标按钮、嵌套交互、菜单/标签壳对比度。
