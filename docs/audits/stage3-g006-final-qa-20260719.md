## 1. 完成范围

1. 完成合同、项目组合框的可访问名称与稳定 ARIA 状态处理；在 Ant Design Vue 4.2.x 下避免关闭下拉时保留悬空的 `aria-controls`、`aria-owns` 和 `aria-activedescendant`。

3. 修复正式业务区的可读性：回款语义色、驾驶舱动作链接、客户发票统计卡与非激活标签的对比度均达到浏览器 axe 验收要求。
4. `StateBlock` 的加载、空态和错误态分别具有可读屏的 `role="status"` / `role="alert"`，驾驶舱补齐成功空响应、缺字段错误、陈旧快照提示和冲突数据状态。
5. 新增可重复执行的 `scripts/verify_g006_final_qa_browser.py`，覆盖核心业务任务、四视口、正式驾驶舱、弱网/空/错误/缺失/陈旧/冲突状态、键盘焦点、减少动效、网络写请求、数据库哈希与 axe。

## 2. 无障碍范围与边界

axe 运行范围固定为每个受验核心页面唯一的 `.pm-workbench-page` 业务内容根节点，执行 WCAG 2 A / AA 规则。该范围包含本次阶段 3 重构的页面标题、查询区、表格、统计卡、驾驶舱和状态区；不将 Vben 全局导航、标签壳、通知铃铛等跨页面框架 DOM 混入业务页面结论。

这个边界不是忽略规则：脚本会先断言页面中恰好存在一个业务根节点；缺失或重复会使验收失败。此前全页扫描已定位并记录框架壳层的无名图标按钮、嵌套按钮和标签壳对比度问题，它们不由本次业务纵向切片引入，仍应在框架升级或单独的全局可访问性任务中治理。

最终业务范围 axe 结果：合同、项目、客户发票、供应商、驾驶舱均为 **0 个 Critical / Serious**。

## 4. 未关闭的存量技术债

### 4.1 全仓 TypeScript

`pnpm -F @vben/web-antd run typecheck` 仍失败。2026-07-19 的最新全仓基线为 **354 项**错误，主要来自既有资产、BPM、WMS、通用组件和插件类型；不能将本次 G006 表述为全仓类型检查通过。阶段 3 改造域（合同、客户应收、供应链、项目、驾驶舱及相关 API 前缀）在同一份 `vue-tsc` 输出中为 **0 项**错误：合同编辑列表索引、只读团队成员 props、合同详情契约、供应商付款列表响应、财务汇总与图表边界均已收口。最新日志位于本地审计资产 `docs/audits/assets/phase3-static-typecheck-after-boundary-fixes.log`；本次新增的 `a11y-select.vue`、`use-accessible-vxe-grid.ts`、状态组件和驾驶舱改动均不在最终错误列表中。

### 4.2 全仓 Lint
