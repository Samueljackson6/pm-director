## ⚠️ 核心原则

**❌ 禁止直接覆盖任何字段**  
**✅ 所有更新必须先读取、再追加、最后更新**

## 强制操作流程

### 步骤 1：读取当前值

```javascript
// 调用 list/get 获取当前记录
const currentRecord = await feishu_bitable_app_table_record({
  action: "list",
  app_token: "HSPXbBE0qaemg4swVAQcchXKnMd",
  table_id: "tblz6ZTwOiMrJLwc",
  filter: {...}
});

// 提取当前值
const currentLastCommunication = currentRecord.fields["最后沟通记录"];
const currentHistory = currentRecord.fields["历史沟通记录"];
```

### 步骤 2：追加到历史字段

```javascript
// 将当前值追加到历史字段
const newHistory = currentHistory + "\n" + currentLastCommunication;

// 或者追加到专门的回款历史字段
const newPaymentHistory = currentPaymentHistory + "\n2026-06-12：50000元";
```

### 步骤 3：更新当前字段

```javascript
// 更新当前字段为新值
await feishu_bitable_app_table_record({
  action: "update",
  app_token: "HSPXbBE0qaemg4swVAQcchXKnMd",
```

## 错误示例 vs 正确示例

### ❌ 错误示例（导致数据丢失）

```javascript
// 直接覆盖，历史数据丢失！
await feishu_bitable_app_table_record({
  action: "update",
  fields: {
    "最后沟通记录": "2026-06-12：新内容"  // ❌ 覆盖了 2026-06-08 的记录
  }
});
```

**后果**：
- 2026-06-08 的沟通记录永久丢失
- 无法追溯历史信息
- 项目进展链条断裂

### ✅ 正确示例

```javascript
// 1. 先读取
const current = await getCurrentRecord();
const oldLastComm = current.fields["最后沟通记录"];  // "2026-06-08：旧内容"
const oldHistory = current.fields["历史沟通记录"];    // "历史记录..."

// 2. 追加历史
const newHistory = oldHistory + "\n" + oldLastComm;
// 结果：历史记录...\n2026-06-08：旧内容

// 3. 更新当前
await updateRecord({
  "历史沟通记录": newHistory,
  "最后沟通记录": "2026-06-12：新内容"
});
```

## 常见错误场景

### 场景 1：更新沟通记录

**错误**：
```
直接更新"最后沟通记录" = "新内容"
→ 历史记录丢失
```

**正确**：
```
1. 读取"最后沟通记录"当前值
2. 追加到"历史沟通记录"
3. 更新"最后沟通记录"为新值
```

### 场景 2：更新回款进度

**错误**：
```
直接更新"回款进度" = "2026-06-12：50000元"
→ 历史回款记录丢失
```

**正确**：
```
1. 读取"回款进度"当前值
2. 追加新记录："当前值\n2026-06-12：50000元"
3. 更新"回款进度"为追加后的值
```

### 场景 3：更新待办事项

**错误**：
```
直接替换"待办事项" = "新待办"
→ 旧待办事项丢失
```

**正确**：
```
1. 读取"待办事项"当前值
2. 添加时间戳标记已完成：✓ 2026-06-08：旧待办（已完成）
3. 追加新待办：\n2026-06-12：新待办
```

## 自动化脚本（推荐）

```python
# 增量更新辅助函数
def incremental_update(record_id, field_name, new_value, history_field=None):
    """
    增量更新飞书表格字段
    
    Args:
        record_id: 记录 ID
        field_name: 当前字段名（如"最后沟通记录"）
        new_value: 新值
        history_field: 历史字段名（如"历史沟通记录"）
    """
    # 1. 读取当前值
    current = get_record(record_id)
    old_value = current.fields.get(field_name, "")
    
    # 2. 追加历史
    if history_field and old_value:
        old_history = current.fields.get(history_field, "")
        new_history = old_history + "\n" + old_value
        update_fields = {history_field: new_history}
    
    # 3. 更新当前
    update_fields[field_name] = new_value
    
    # 4. 执行更新
    update_record(record_id, update_fields)
```
