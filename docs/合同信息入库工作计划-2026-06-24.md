## 二、工作流程（每个合同）

### 2.1 识别阶段（约5分钟/合同）

**步骤**：
1. 下载合同PDF到本地
2. 使用PaddleOCR + PP-StructureV3识别
3. 提取关键字段：
   - 合同编号
   - 项目名称
   - 甲乙方信息
   - 签订日期
   - 合同金额
   - 付款计划
   - 交付物

### 2.2 分析阶段（约10分钟/合同）

**步骤**：
1. 分析合同结构：
   - 合同类型（科技项目/工程/采购/服务）
   - 条款结构
   - 表格格式
   
2. 对照管理规范：
   - 现有项目管理表格字段是否完整？
   - 是否需要新增字段？
   - 数据存放位置是否合理？
   - 数据关联设计是否合理？

3. 记录问题和建议

### 2.3 入库阶段（约5分钟/合同）

**步骤**：
1. 字段映射（合同字段 → 表格字段）
2. 数据校验（格式、逻辑、完整性）
3. 填充到飞书多维表格
4. 记录入库日志

## 四、关键决策点

### 4.1 合同类型分类

**识别方式**：根据合同名称和内容判断

| 合同类型 | 特征 | 示例 |
|---------|------|------|
| **科技项目** | 含"研究"、"技术"、"开发" | 变电站保护定值自动核对技术研究 |
| **工程服务** | 含"维护"、"监测"、"测绘" | 水电站大坝形变观测技术服务 |
| **采购合同** | 含"采购"、"供货"、"设备" | （待识别） |
| **咨询服务** | 含"咨询"、"服务" | 区域变电站预警能力提升咨询服务 |

### 4.2 字段映射规则

**核心字段映射**：

| 合同字段 | 表格字段 | 转换规则 |
|---------|---------|---------|
| 合同编号 | 项目编号 | 直接映射 |
| 项目名称 | 项目名称 | 直接映射 |
| 甲方 | 客户名称 | 直接映射 |
| 乙方 | - | 固定值（振华永创） |
| 签订日期 | 合同签订日期 | 标准化为YYYY-MM-DD |
| 合同金额 | 项目金额 | 单位：元 |
| 付款计划 | 回款进度 | 增量记录 |

### 4.3 数据校验规则

**必填字段**：
- ✅ 项目名称
- ✅ 客户名称
- ✅ 项目金额
- ✅ 合同签订日期

**格式校验**：
- 日期：YYYY-MM-DD
- 金额：数字，单位元
- 合同编号：符合编码规则

**逻辑校验**：
- 回款金额 ≤ 项目金额
- 签订日期 ≥ 2020-01-01
- 预计交付日期 ≥ 签订日期

## 六、风险管理

### 6.1 识别失败风险

**风险**：OCR识别准确率低，关键字段缺失

**应对**：
- 使用PP-StructureV3提高准确率
- 人工辅助校对
- 记录识别失败案例

### 6.2 字段映射风险

**风险**：合同字段与表格字段不匹配

**应对**：
- 灵活映射，记录映射规则
- 及时更新表格字段设计
- 预留扩展字段

### 6.3 数据质量风险

**风险**：数据不完整、格式不统一

**应对**：
- 建立数据校验规则
- 定期数据质量检查
- 增量维护规范
