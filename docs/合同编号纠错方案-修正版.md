## 一、原方案问题

### 1.1 原方案（强制格式）

```python
def correct_contract_number(text: str) -> str:
    """修正合同编号（强制格式）"""
```

**问题**：
- ❌ 过于僵化，无法适应不同类型合同
- ❌ 不同公司/项目合同编号格式不同

## 二、修正方案（模式识别+统计分析）

### 2.1 核心思路

**不强制格式，而是通过多样本分析总结规律**

#### 步骤1：识别多份合同，收集合同编号样本

```python
contract_numbers = [
    "SGSCCD00JYJS2401318",  # 虚拟电厂合同

    # ... 更多样本

```

#### 步骤2：分析样本规律

**方法**：
- 统计每个位置字符分布（字母/数字比例）
- 识别常见混淆模式（0/O、I/J等）
- 建立纠错概率模型

```python
from collections import Counter

def analyze_contract_numbers(samples: List[str]) -> Dict:
    """分析合同编号样本规律"""
    
    analysis = {
        "length_dist": Counter(len(s) for s in samples),
        "position_char_type": {},  # 每个位置的字符类型分布
        "common_patterns": [],      # 常见模式
    }
    
    # 分析每个位置
    for i in range(max(len(s) for s in samples)):
        chars_at_pos = [s[i] if i < len(s) else None for s in samples]
        char_types = Counter(
            'digit' if c.isdigit() else 'letter' if c.isalpha() else 'other'
            for c in chars_at_pos if c
        )
        analysis["position_char_type"][i] = char_types
    
    return analysis
```

#### 步骤3：基于统计的纠错

```python
def correct_contract_number_v2(text: str, analysis: Dict) -> str:
    """基于统计分析的纠错"""
    
    # 1. 提取疑似合同编号
    candidates = re.findall(r'[A-Z0-9/]{10,30}', text)
    
    # 2. 对每个候选进行纠错
    corrected = []
    for candidate in candidates:
        # 根据统计规律判断每个位置应该是字母还是数字
        corrected_str = ""
        for i, char in enumerate(candidate):
            if i in analysis["position_char_type"]:
                expected_type = analysis["position_char_type"][i].most_common(1)[0][0]
                
                # 纠错逻辑
                if expected_type == 'digit' and char.isalpha():
                    # 应该是数字，但识别为字母
                    corrected_char = char_confusion_map.get(char, char)
                elif expected_type == 'letter' and char.isdigit():
                    # 应该是字母，但识别为数字
                    corrected_char = digit_to_letter_map.get(char, char)
                else:
                    corrected_char = char
                
                corrected_str += corrected_char
            else:
                corrected_str += char
        
        corrected.append(corrected_str)
    
    return corrected[0] if corrected else text

# 字符混淆映射
char_confusion_map = {
    'O': '0', 'o': '0',
    'I': '1', 'i': '1', 'l': '1',
    'S': '5', 's': '5',
    'B': '8', 'b': '8',
    'Z': '2', 'z': '2',
}

digit_to_letter_map = {
    '0': 'O',
    '1': 'I',
    '5': 'S',
    '8': 'B',
    '2': 'Z',
}
```

### 3.3 纠错规则生成

**基于分析结果自动生成纠错规则**：

```python
def generate_correction_rules(analysis: Dict) -> List[Dict]:
    """根据分析结果生成纠错规则"""
    
    rules = []
    
    for pos, char_type_dist in analysis["position_char_type"].items():
        # 如果某位置90%以上是数字，则该位置应强制为数字
        if char_type_dist.get('digit', 0) / sum(char_type_dist.values()) > 0.9:
            rules.append({
                "position": pos,
                "expected_type": "digit",
                "confidence": char_type_dist.get('digit', 0) / sum(char_type_dist.values())
            })
        # 如果某位置90%以上是字母，则该位置应强制为字母
        elif char_type_dist.get('letter', 0) / sum(char_type_dist.values()) > 0.9:
            rules.append({
                "position": pos,
                "expected_type": "letter",
                "confidence": char_type_dist.get('letter', 0) / sum(char_type_dist.values())
            })
    
    return rules
```

## 四、方案优势

| 维度 | 强制格式方案 | 统计分析方案 |
|------|------------|------------|
| **适应性** | ❌ 只适用于特定格式 | ✅ 适用于多种格式 |
| **准确性** | ⚠️ 可能误纠 | ✅ 基于统计规律 |
| **可维护性** | ❌ 需手动修改代码 | ✅ 自动更新规则 |
| **可扩展性** | ❌ 新格式需改代码 | ✅ 新样本自动学习 |
