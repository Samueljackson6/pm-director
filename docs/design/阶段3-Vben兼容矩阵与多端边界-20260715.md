# 阶段3：Vben 兼容矩阵与多端边界

> 日期：2026-07-15
> 结论：旧 Demo 的信息形式大多可实现，但必须在真实 Vben 组件、动态菜单、主题和路由契约中重写；Vben 组件不可直接复用于小程序和 App。

## 一、当前框架事实

- 主前端：`ui-vben/apps/web-antd`。
- 框架：Vben Admin 5 + Vue 3 + Ant Design Vue。
- 权限模式：`accessMode: 'backend'`。
- 菜单真值：`backend/routers/auth.py::vben_permission_info()`。
- 表格：`#/adapter/vxe-table` 的 `useVbenVxeGrid`。
- 图表：`@vben/plugins/echarts` 的 `EchartsUI/useEcharts`。
- API：`#/api/request` 的 `requestClient`。
- 当前依赖快照：Ant Design Vue 4.2.6、ECharts 6.1.0、VxeTable 4.19.23。

## 二、组件兼容矩阵

| 设计模块 | 生产组件 | 兼容结论 | 实施约束 |
|----------|----------|----------|----------|
| 应用导航 | Vben `BasicLayout` + 后端菜单 | 兼容 | 禁止自建第二套侧栏、顶栏和移动遮罩 |
| 页面根容器 | `Page` | 兼容 | Drawer `appendToMain` 时使用 `auto-content-height` |
| 合同/任务列表 | `useVbenVxeGrid` | 兼容 | 远程分页、筛选回填、formatter/cellRender；禁用原生静态表格替代生产列表 |
| 筛选与录入 | `useVbenForm` | 兼容 | URL Query 与后端参数同源；高频/低频条件分层 |
| 详情与页签 | Page + Ant Card/Tabs/Descriptions | 兼容 | Tab 写入 Query；直达详情有返回兜底 |
| 高层履约阶段 | Ant Steps | 有条件兼容 | 只表达高层状态；科研 N:1 和复杂触发关系不能强画成线性步骤 |
| 证据/操作事件 | Ant Timeline | 兼容 | 与履约阶段分开，展示来源、抽取、核验和修订历史 |
| 数据核验抽屉 | `useVbenDrawer` | 兼容 | connectedComponent、destroyOnClose、lock 防重复提交 |
| 加载/错误/空态 | 项目 `StateBlock` | 兼容 | 必须传真实三态，不保留 Demo 的评审状态选择器 |
| 趋势/关系图 | `EchartsUI/useEcharts` | 兼容 | 只展示有来源的指标，使用主题令牌并设置容器高度 |
| 菜单搜索 | Vben GlobalSearch | 兼容菜单 | 内置只搜菜单；业务实体搜索需单独 API，不能争用同一 Ctrl+K |
| 主题 | Vben Preferences + HSL Token | 兼容 | 默认浅色，不保留独立 `data-theme` 和大块固定深色 |
| 响应式 Web | Vben 壳 + 页面断点 | 部分兼容 | 密集表格、详情双栏和 Steps 仍需逐页设计 |
| 小程序/App | 共享 API/状态/Token | 组件不兼容 | 不复制 Vben DOM 和桌面菜单 |

## 三、Vben 原生评审 Demo 接入

本轮不修改后端菜单，也不增加生产数据接口。评审模式复用现有规范路由：

```text
/dashboard/overview/index?review=phase3
```

页面在真实 Vben 壳中加载，使用 Page、VxeGrid、Ant Design Vue 与 Vben 主题令牌。评审数据是明确标注时间的只读快照和脱敏业务样例，不声称实时、不执行写操作。

## 四、视觉方向

**方向：浅色精密仪器 / 编辑式业务工作台。**

- 画布以白、雾灰、浅青为主，不使用整列深蓝或全屏暗色卡片。
- 唯一识别元素是“证据轨道”：用细线、节点和小面积青/琥珀色表达 PDF→DOCX→抽取→核验。
- 数字不使用无依据的健康分；优先显示来源、时点、缺失和下一动作。
- 动效仅用于页签切换、轨道节点和状态反馈，160–240ms，并支持 reduced-motion。
- 状态不只依赖颜色，始终配合文字、图标与动作。

## 五、Web、小程序和 App 分工

| 端 | 主要用户 | 核心任务 | 不承担 |
|---|---|---|---|
| Web | 合同管理员、财务、数据管理员、项目总监 | 批量导入、字段对比、全文核验、财务对账、供应商关联、复杂报表、权限配置 | 高频现场拍摄、长期离线 |
| 小程序 | 项目经理、现场人员、管理层 | 今日待办、节点提醒、拍照上传、进度填报、轻量验收、快速审批 | 大表格、全文条款对比、批量映射 |
| App | 弱网/长期现场人员 | 离线草稿、断点续传、完整现场验收、设备能力、稳定推送 | 复制桌面后台菜单 |

小程序首期只面向内部人员。客户/供应商门户需要租户隔离、权限和审计成熟后另行立项。

## 六、多端共享领域对象

共享：Contract、ContractArtifact、ContractAlias、Project、Stage、Deliverable、AcceptanceRecord、PaymentPlan、CustomerInvoice、CustomerReceipt、SupplierContract、SupplierInvoice、SupplierPayment、ImportBatch、SourceRecord、DataIssue、ReviewTask、ApprovalTask、Notification。

不共享：Vben 页面组件、桌面大表格、三端各自的金额公式、权限规则和状态判断。

## 七、移动能力缺口

首批后端能力建议：

```text
/api/v1/artifacts/upload-sessions
/api/v1/import-batches
/api/v1/data-issues
/api/v1/review-tasks
/api/v1/field-acceptances
/api/v1/approval-tasks
/api/v1/mobile/sync/pull
/api/v1/mobile/sync/push
/api/v1/notification-subscriptions
```

关键非功能要求：客户端幂等键、加密草稿、同步游标、乐观锁、冲突提示、上传分片、断点续传、文件哈希去重、最小化离线缓存和完整审计。

## 八、生产风险

1. 关闭 Vben 标签栏会同时影响 KeepAlive，筛选/详情状态必须改由 Query 或 Store 恢复。
2. 只修改 `routes/modules` 不会在后端模式生成菜单。
3. 当前项目入口缺失、供应商详情自重定向、看板双导航仍是独立生产问题。
4. `inbound` 历史语义未迁移前，供应商发票页面不得接现有数据。
5. Vben 响应式壳不等于移动业务设计，小程序/App 需独立验收。
