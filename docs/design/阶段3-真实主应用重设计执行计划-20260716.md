# 阶段 3 真实主应用重设计执行计划

> 交接说明（2026-07-19）：Wave 1 已完成，Wave 2–3 正在 G003 中收口，Wave 4–5 尚未开始。后续实施、风险和验证口径统一从 [阶段 3 交接入口](../handoff/README.md) 继续。

## 目标

在不更换 Vben 技术栈、不修改业务数据库、不虚构未实现能力的前提下，重构真实主应用的菜单、驾驶舱及全部核心页面。每个生产改动先建立失败测试，再实现并验证；浏览器验收覆盖 1440、1024、768、390 四个宽度。

## 目标菜单树

```text
经营工作台
└─ 经营与任务驾驶舱

合同履约
├─ 合同台账
└─ 项目台账

客户应收
├─ 销项发票
└─ 客户回款

供应链
├─ 供应商档案
├─ 供应商收票
└─ 供应商付款
```

现有 URL 尽量保持兼容，详情页作为隐藏子路由归属相应业务组。角色化内容不等于已具备 RBAC；当前仍按管理员可见范围实施。

## Wave 1：规范、菜单与路由基线

1. 建立菜单契约测试，覆盖四组名称、排序、子路由和隐藏详情归属。
2. 建立路由冒烟测试，先复现项目页 404 和供应商详情 Legacy 冲突。
3. 调整 `backend/routers/auth.py` 的动态菜单，恢复项目列表/详情正式路由。
4. 清理供应商详情自重定向与错误的项目 Legacy 详情重定向。
5. 固化详情进入/返回来源契约，保证 query、筛选和分页可恢复。
6. 浏览器验证四组入口、全部列表/详情路由及无 404。

停止条件：任何路由仍 404、组件名无法由动态路由解析、发布覆盖层与主源码无法确定唯一真相。

## Wave 2：超长详情页特征锁定与拆分

1. 为 `views/contracts/detail.vue` 建立请求、渲染、事件、空态特征测试。
2. 在特征测试全绿后拆出合同头部/事实链、履约、财务、文件与关联对象组件。
3. 父页只保留数据加载、状态和页面编排，单个纯逻辑文件不超过项目规则上限。
4. 为 `views/suppliers/detail/index.vue` 建立同级特征测试。
5. 拆出供应商档案、合作关系、工商信息、收票与付款状态组件。
6. 对比拆分前后 DOM、网络请求和交互，不夹带业务修复。

停止条件：请求参数、次数、页面状态或现有可用交互发生非预期漂移。

## Wave 3：实体标识、金额单位与业务语义

1. 修复供应商列表 `supplier_id` 到详情聚合 API 的标识链路，企查查查询只使用真实信用代码。
2. 客户发票页移除供应商发票 Tab 和客户回款伪发票语义，金额统一为元存储、万元展示。
3. 客户回款页修复当前页汇总冒充全量、已匹配金额未赋值和状态字典错位。
4. 自动匹配改为不可直接触发写库的预览/确认边界；在缺少安全后端契约时禁用并解释。
5. 供应商收票不得读取历史客户回款；数据源未建立时展示明确空态与迁移说明。
6. 供应商付款展示前置条件和来源状态，不伪造 0、合计或关联发票。
7. 驾驶舱 API/类型增加口径、来源、覆盖范围、核验状态和真实数据截止时间；不再把请求时间称为更新时间。

停止条件：修复需要清洗或迁移数据库、付款来源无法确认、会产生未授权写请求。

## Wave 4：Vben 页面基础与全域页面重设计

1. 落地浅色“电网白图”语义 Token，并映射到 Vben 主题变量。
2. 修复 `CellRouterLink` 语义、焦点环和硬编码颜色。
3. 所有核心列表/详情使用 Vben `Page`，移除整屏灰渐变第二壳层。
4. 生产列表统一到 `useVbenVxeGrid`；只读嵌入表也必须有显式详情入口和可访问摘要。
5. 查询表单、写入 Modal、核验 Drawer 和 `StateBlock` 形成统一模板。
6. 逐域改造合同履约、客户应收、供应链所有列表和详情页。
7. 移动宽度使用任务卡或保留关键事实的表格策略，不通过隐藏金额、状态和期限完成适配。
8. 落地 reduced-motion、键盘、焦点、对比度、触控目标与图表摘要。

停止条件：迁移改变分页、筛选、权限或数据写入语义；390px 主流程仍需页面级横向滚动。

## Wave 5：角色任务驾驶舱与整体验收

1. 移除真实驾驶舱对被否决 Phase3Review Demo 的条件依赖。
2. 重构为综合、经营、项目、财务、数据核验五个视角；每个视角有不同决策内容与动作。
3. 首屏顺序固定为待办/缺口、风险/到期、数据可信度、角色摘要、最近变化。
4. 指标采用“值、状态、截止时间、来源、已核验/总数、解释、动作”的统一契约。
5. 所有告警目标路由与查询参数由目标页真实消费，消除双跳和无效筛选。
6. 完成 1440/1024/768/390 视觉、键盘、axe、网络、返回上下文和错误/空态验收。
7. 由独立 reviewer 检查业务目标、代码质量、安全、Vben 兼容、视觉一致性和无障碍；所有 blocker 修复后重新评审。

停止条件：关键任务不可达、存在伪造数字、Critical/Serious 无障碍问题、数据库差异非零或 reviewer 未无条件通过。
