## 一、任务范围

P0-4 包含三部分工作，**必须一起完成**：

### 4.1 前端金额组件改造（因 API 已改为输出元）

**背景**：后端 P0-2 已完成所有金额字段从万元→元的转换。前端代码目前按"万元"展示，如果直接打开页面，金额会显示 10000 倍。

**改动要求**：
- 所有从后端 API 获取的金额字段，在展示层统一除以 10000 转为万元显示
- 建议创建统一的金额格式化函数 `formatWanYuan(value)`，集中管理转换逻辑
- 涉及页面：经营工作台、合同台账、合同详情、项目台账、项目详情、客户发票、客户回款、供应商、财务汇总
- **不改动**：`invoices.amount` 和 `receipts.amount` 已是元，前端展示时也要 ÷10000

**影响面**：
| 页面 | 涉及组件 |
|------|---------|
| 经营工作台 Dashboard | KPI 卡片、图表 tooltip、pending tasks |
| 合同台账列表 | contract_amount、invoice_total、payment_total 列 |
| 合同详情页 | 合同金额卡片、付款计划表、财务摘要 |
| 项目台账列表 | total_contract_amount 列 |
| 项目详情页 | 总金额卡片、付款进度 |
| 客户发票列表/详情 | amount 列、汇总卡片 |
| 客户回款列表/详情 | amount 列、匹配状态 |
| 供应商列表/详情 | total_contract_amount、付款记录 |
| 财务汇总 | 所有金额字段 |
