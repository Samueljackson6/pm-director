# 贡献约束

> 本文档定义 pm-director 项目的代码贡献约束，包括路由规范、返回契约、命名规范等。
> 来源：系统全量审计报告 `docs/audits/系统全量审计与改造基线-2026-07-13.md`

## 1. 路由规范

### 1.1 核心原则

**后端菜单是路由的唯一真值**

本项目使用 `accessMode: 'backend'`，所有业务路由必须通过后端 API (`backend/routers/auth.py`) 定义。

```
┌─────────────────────────────────────────────────────────────────┐
│                      路由真值层级                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   后端 auth.py menus ──────────────────────────> 唯一真值       │
│         │                                                       │
│         ▼                                                       │
│   前端动态路由生成 ───────────────────────────> 业务路由        │
│                                                                 │
│   前端 routes/modules/ ───────────────────────> 仅允许重定向    │
│         │                                                       │
│         └── 只允许 name.startsWith('Legacy') 的重定向路由       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 新增页面流程

```
1. 在 views/ 下创建 .vue 文件
   └─ 例：views/suppliers/list/index.vue

2. 在 auth.py 的 menus 中添加菜单项
   └─ 例：
      {
          'name': '供应商列表',
          'path': 'list',                    # 相对路径！
          'component': '/suppliers/list/index',
          'componentName': 'SupplierList',
          'visible': True,
      }

3. 验证路由
   └─ curl http://localhost:18080/system/auth/get-permission-info
```

### 1.3 路径格式规范

| 场景 | 正确 ❌ | 错误 ✅ |
|------|--------|--------|
| 一级模块路径 | `/suppliers` | `suppliers` |
| 子菜单路径 | `list`（相对） | `/suppliers`（绝对） |
| 组件路径 | `/suppliers/list/index` | `suppliers/list/index.vue` |

**关键规则**：

- 一级模块使用绝对路径（如 `/suppliers`）
- 子菜单使用相对路径（如 `list`、`detail`）
- 组件路径以 `/` 开头，不包含 `views/` 前缀和 `.vue` 后缀

### 1.4 禁止的路由操作

| 禁止操作 | 原因 | 替代方案 |
|---------|------|---------|
| 在前端 `routes/modules/` 中添加业务路由 | 绕过后端菜单控制 | 在 `auth.py` 中定义 |
| 使用绝对路径作为子菜单路径 | 导致路径拼接错误（`/suppliers/suppliers`） | 使用相对路径 |
| 直接访问静态路由定义的页面 | 菜单不可见不等于路由不可达 | 确保页面在 `auth.py` 中定义 |

## 2. 返回契约规范

### 2.1 问题背景

审计发现详情页返回进入"用户没见过"的页面：

| 详情页 | 原返回目标 | 问题 |
|--------|-----------|------|
| 项目详情 | `ProjectList` | ❌ 返回隐藏的旧页面 |
| 客户发票详情 | `InvoiceList` | ❌ 返回旧发票页 |
| 供应商详情 | `SupplierList` | ⚠️ 路径解析错误 |

### 2.2 返回契约原则

```
┌─────────────────────────────────────────────────────────────────┐
│                      返回契约原则                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 来源优先                                                    │
│     └─ 列表进入详情时携带 from 参数                              │
│     └─ 详情页优先使用 router.back() 或 from 参数返回            │
│                                                                 │
│  2. 规范兜底                                                    │
│     └─ 无可信来源时，回到模块规范列表                            │
│     └─ 使用 router.push({ name: 'ModuleNameList' })             │
│                                                                 │
│  3. 旧路径隔离                                                  │
│     └─ 旧路径只做单向重定向                                      │
│     └─ 新页面不得主动跳回旧路由                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 实现示例

**推荐写法**：

```typescript
// 列表页：进入详情时携带来源
function goToDetail(id: string) {
  router.push({
    name: 'CustomerInvoiceDetail',
    query: { id },
    state: { from: 'CustomerInvoices' }
  });
}

// 详情页：返回逻辑
function goBack() {
  // 优先返回来源
  if (router.options.history.state.back) {
    router.back();
  } else {
    // 兜底：回到规范列表
    router.push({ name: 'CustomerInvoices' });
  }
}
```

**禁止写法**：

```typescript
// ❌ 固定跳旧列表
function goBack() {
  router.push({ name: 'InvoiceList' }); // 旧路由
}
```

### 2.4 规范路由命名对照表

| 模块 | 列表路由名 | 详情路由名 | 后端 componentName |
|------|-----------|-----------|-------------------|
| 合同 | `ContractList` | `ContractDetail` | `ContractList` / `ContractDetail` |
| 客户发票 | `CustomerInvoices` | `CustomerInvoiceDetail` | `CustomerInvoices` / `CustomerInvoiceDetail` |

## 4. 金额单位规范

### 4.1 问题背景

审计发现金额单位混标：

- 录入字段为"元"
- 列表/KPI 原值标"万元"
- 导致金额放大 10,000 倍展示

### 4.2 规范要求

| 层级 | 要求 |
|------|------|
| API Schema | 必须明确金额单位（`amount_yuan` 或 `amount_wan`） |
| 数据库字段 | 必须在字段名或注释中标注单位 |
| 前端展示 | 必须在列标题或数值旁标注单位 |

**示例**：

```typescript
// ✅ 正确：明确单位
const columns = [
  { field: 'amount', title: '金额（万元）', formatter: formatMoney }
];

// ❌ 错误：单位不清
const columns = [
  { field: 'amount', title: '金额' } // 是元还是万元？
];
```

## 5. 数据契约规范

### 5.1 问题背景

审计发现：

- SQLite `foreign_keys = 0`（外键关闭）
- 167 条外键违规记录
- 数据关系不可信

### 5.2 规范要求

| 操作 | 要求 |
|------|------|
| 新建表 | 必须定义主键 |
| 关联字段 | 必须定义外键约束 |
| 迁移脚本 | 必须包含 dry-run 和回滚脚本 |
| 数据清洗 | 必须先备份再操作 |

### 5.3 外键违规分类

| 分类 | 处理方式 |
|------|---------|
| 可自动修复 | 确认后执行修复脚本 |
| 需业务确认 | 记录问题，等待业务决策 |
| 应归档 | 移动到归档表 |

## 6. 代码风格约束

### 6.1 VxeTable 使用规范

**正确写法**：

```vue
<vxe-table :data="list">
  <vxe-column field="name" title="名称" :formatter="formatName" />
</vxe-table>
```

**禁止写法**：

```vue
<!-- ❌ 禁止使用 slot 访问 row -->
<vxe-table :data="list">
  <vxe-column field="name" title="名称">
    <template #default="{ row }">
      {{ row.name }}
    </template>
  </vxe-column>
</vxe-table>
```

### 6.2 API 调用规范

**正确写法**：

```typescript
import { requestClient } from '#/api/request';

const data = await requestClient.get('/api/suppliers');
// requestClient 自动解包 data
```

### 6.3 状态管理规范

使用 `StateBlock` 统一处理加载、错误和空态：

```vue
<state-block :loading="loading" :error="error" @retry="loadData">
  <!-- 正常内容 -->
</state-block>
```

## 7. 旧路由迁移规范

### 7.1 迁移流程

```
1. 确认新路由已定义
   └─ 在 auth.py 中定义新路由

2. 添加重定向
   └─ 在 routes/modules/legacy-redirects.ts 中添加

3. 更新所有引用
   └─ Grep 搜索旧路由名/路径
   └─ 更新为新路由

4. 设置退役日期
   └─ 在 docs/规范路由清单-*.md 中记录

5. 验证
   └─ 测试旧路径重定向
   └─ 测试新路径正常工作
```

## 8. 检查清单

### 8.1 新增页面前检查

- [ ] 是否在 `auth.py` 中定义了菜单项？
- [ ] 路径是否使用相对路径（子菜单）？
- [ ] 组件路径是否正确（以 `/` 开头，无 `.vue` 后缀）？
- [ ] componentName 是否唯一？
