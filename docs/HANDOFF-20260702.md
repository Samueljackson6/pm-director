## 三、关键架构决策

| 决策 | 说明 |
|------|------|
| **Vben 框架分离** | Vben Admin v5 是独立项目（npm create 获取），不是 pm-director 子目录。pm-director 只包含 UI 自定义代码 (`ui/`) |
| **SQLite 只读** | 后端使用 SQLite 只读 API，数据通过外部工具写入 |
| **ignoreAccess** | 所有自定义路由均设 `ignoreAccess: true`，绕过 Vben 登录认证 |
| **Vben 响应格式** | 后端包装 `{code: 0, data: {...}}` 以兼容 Vben requestClient |
| **Docker 测试** | 使用 Docker 容器跑测试，用完即停。`docker compose up -d` / `docker compose down` |
| **RuoYi 分离** | RuoYi Office Vben (端口 5666) 是独立项目，与 pm-director 无关 |

## 十、自动化流水线体系 (v2.0 — 2026-07-02 Session 4)

### 完整流程

```
开发 → lint检查 → git commit + push → 
  GitHub Actions CI (自动) → 
  一键部署到236 (auto-publish.sh) → 
  验证 (HTTP 200) → 审计 (完成度≥70%)
```

### 文件体系

| 文件 | 用途 |
|------|------|
| `docs/工作流规范.md` | **总纲** — 开发→测试→提交→部署→验证→审计六阶段 |
| `docs/约束与规范.md` | **铁律** — 强制约束、违规记录、执行规则 |
| `docs/部署规范.md` | 部署细节（v2.0 自动版，含 systemd/CI/故障处理） |
| `docs/Vben开发规范.md` | Vben 前端开发规范 |
| `.github/workflows/ci.yml` | GitHub Actions CI（Python lint + UI验证 + 提交信息检查） |
| `scripts/auto-publish.sh` | 一键发布脚本（完整流水线） |
| `scripts/deploy-236.sh` | 236 部署脚本（backend\|frontend\|all\|verify） |
| `systemd/pm-backend.service` | 后端 systemd 服务 |
| `systemd/pm-frontend.service` | 前端 systemd 服务 |
| `systemd/install.sh` | 236 一键安装 systemd（需 sudo 密码） |

### 局限性
- 236 无公网 IP → GitHub Actions 无法直接 SSH → 实际部署通过 auto-publish.sh 本地触发
- systemd 安装需 sudo 密码 → 建议配置 sudoers NOPASSWD 或用户手动运行 install.sh
- 当前临时用 nohup 管理服务 → 建议尽快切换到 systemd

### 使用方式

```bash
# 1. 一键发布（完整流水线，从本地执行）
bash scripts/auto-publish.sh

# 2. 仅部署（代码已推送后）
bash scripts/auto-publish.sh deploy

# 3. 仅验证/审计
bash scripts/auto-publish.sh verify
bash scripts/auto-publish.sh audit

# 4. 在236上直接部署
ssh 236 "bash scripts/deploy-236.sh all"

# 5. 安装 systemd 服务（需sudo，在236上运行一次）
ssh 236 "bash systemd/install.sh"
```
