# 问题3修复方案：服务类合同付款数据提取

## 一、问题分析

### 现状
- 23个服务类合同付款总额为0
- OCR中有付款条款，但未被提取为结构化付款记录

### 根因
服务类合同的付款条款格式与科研类不同：
- 科研类：明确的付款计划表格，每行有具体金额
- 服务类：通常是"按进度付款"或"验收后付款"，没有明确的金额表格

### 服务类合同付款模式分类

#### 类型A: 一次性付款
**特征**: "一次性支付"、"支付合同金额100%"

**示例**: ZH02-202508013
- 条款: "技术服务内容全部完成，经甲方验收合格...支付合同金额100%"
- 修复方案: 创建1条付款记录，金额=合同金额×100%

#### 类型B: 按进度付款（有比例）
**特征**: "完成80%后支付进度款"、"验收合格后支付剩余"

**示例**: ZH02-202406004
- 条款: "(1)验收合格后支付80%进度款 (2)交齐资料后支付剩余"
- 修复方案: 创建2条付款记录
  - 第一期: 合同金额×80%
  - 第二期: 合同金额×20%

**示例**: ZH02-202508019
- 条款: "进度款最高80%...验收后支付97%，剩余3%质保金"
- 修复方案: 创建3条付款记录
  - 进度款: 合同金额×80%
  - 验收款: 合同金额×17%
  - 质保金: 合同金额×3%

#### 类型C: 按进度付款（无比例）
**特征**: "按进度付款"但没有具体比例

**示例**: 部分合同条款模糊，需要人工确认

## 二、修复方案

### 方案1: 自动提取（适用于明确比例的合同）

**提取规则**:
```python
# 识别付款模式
patterns = {
    "一次性": r"一次性.*支付.*100%",
    "80%进度款+剩余": r"支付.*80%.*进度款.*剩余",
    "验收后付款": r"验收合格.*支付",
}

# 根据模式创建付款记录
def create_payments(contract_id, contract_amount, payment_pattern):
    if payment_pattern == "一次性":
        return [(contract_amount, "一次性支付")]
    elif "80%进度款" in payment_pattern:
        return [
            (contract_amount * 0.8, "完成进度并验收合格后支付"),
            (contract_amount * 0.2, "交齐竣工资料后支付")
        ]
    # ... 其他模式
```

### 方案2: 手动补充（适用于复杂/模糊合同）

对于无法自动解析的合同，需要人工读取OCR原文并手动补充。
