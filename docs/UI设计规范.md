# UI 设计规范

> 版本：2.0 设计基线
>
> 更新日期：2026-07-15
>
> 适用应用：`ui-vben/apps/web-antd`
> 关联信息架构：[阶段 3 信息架构与规范路由](./design/阶段3-信息架构与规范路由-20260715.md)

## 1. 产品与设计目标

PM Director 是面向项目总监、项目经理和财务人员的合同履约管理系统。
界面的单一工作目标是帮助用户判断下一步要处理什么，并在合同、项目、交付、开票、回款和付款之间可靠移动。

设计优先级如下：

1. 任务可完成。
2. 位置和返回目标可理解。
3. 金额、状态、风险和按钮诚实。
4. 用尽可能少的界面元素表达业务关系。
5. 在真实数据量和设备上保持可读、可访问和高性能。

## 2. 设计方向与校准

### 2.1 第一版方向

第一版方向是“专业运营控制台”：品牌蓝、白色卡片、KPI、图表和表格。
这与当前页面相似，也容易退化为任何后台都能使用的通用模板。

### 2.2 校准后的方向

最终方向是“履约调度台”：

- 全局结构使用克制的单侧栏和清晰页面标题。
- 驾驶舱先显示下一步行动，再显示指标和图表。
- 列表减少卡片包裹，突出筛选、对象链接和数据密度。
- 详情页使用一条“履约链”串联合同、项目、交付、开票和回款。
- 颜色只表达业务语义，不作为装饰。

“履约链”是唯一允许明显体现产品个性的元素。
其他区域保持安静、平直和可扫描，避免渐变背景、装饰性大数字、Emoji 和多层圆角卡片。

## 3. 全局壳层

### 3.1 Vben 布局

- `app.layout` 使用 `sidebar-nav`。
- 顶栏只保留全局搜索、刷新、主题、帮助和用户工具。
- 顶栏不显示一级业务菜单。
- `tabbar.enable` 设为 `false`，关闭全局页面标签栏。
- 面包屑启用，不显示图标。
- 页脚关闭。
- 移动端使用 Vben 自带侧栏抽屉。

### 3.2 页面容器

业务页面以 Vben `Page` 作为顶层容器：

- `title`：当前页面或对象名称。
- `description`：页面任务、数据时间和单位。
- `extra`：当前页面的主要操作。
- 内容区不得再重复渲染一个相同标题的卡片。

页面水平留白：

| 视口 | 内容边距 |
|---|---:|
| 小于 768px | 12px |
| 768px 至 1199px | 16px |
| 1200px 及以上 | 24px |

## 4. 色彩系统

### 4.1 核心色板

只允许以下六个产品级核心色直接进入集中式 Token。
业务组件不得出现新的十六进制颜色。

| Token | 名称 | 色值 | 用途 | 白底对比度 |
|---|---|---|---|---:|
| `--pm-brand` | 调度蓝 | `#1D4ED8` | 主操作、链接、当前项 | 6.70:1 |
| `--pm-ink` | 电网墨 | `#172B4D` | 主要文本、标题 | 14.10:1 |
| `--pm-success` | 验收绿 | `#067647` | 已完成、已回款、已付款 | 5.69:1 |
| `--pm-warning` | 预警琥珀 | `#934A00` | 待交付、待回款、待付款 | 6.52:1 |
| `--pm-danger` | 故障红 | `#B42318` | 逾期、异常、失败 | 6.57:1 |
| `--pm-canvas` | 工作台雾 | `#F5F7FA` | 页面背景 | 不作为文字色 |

信息类青色、次要文字、边框和浅色背景优先使用 Ant Design/Vben 主题 Token。
图表需要额外序列色时，必须在图表主题文件集中定义，单个图表不得自建色板。

### 4.2 语义规则

- 颜色必须同时配合文字、图标或形状，不能作为唯一状态信号。
- 完成状态使用绿，执行中和链接使用蓝，等待处理使用琥珀，异常和逾期使用红。
- “未评估”“待核验”“未知”使用中性灰，不使用绿色。
- 大面积背景使用主题浅色 Token，语义色主要用于文字、图标、边框和小型状态块。
- 深色模式使用 Ant Design 暗色算法映射，逐项重新验证对比度，不直接复用浅色背景。

### 4.3 色彩数量上限

- 产品核心色：6 个。
- 单个图表序列：最多 6 色。
- 单个页面可见的语义强调色：最多 4 色。
- 单个业务组件内的字面颜色值：0 个。

## 5. 字体与数字

### 5.1 字体角色

| 角色 | 字体栈 | 用途 |
|---|---|---|
| 中文界面 | `"HarmonyOS Sans SC", "Microsoft YaHei UI", "Noto Sans CJK SC", sans-serif` | 标题、正文、控件 |
| 数字与编号 | `"Roboto Mono", "Cascadia Mono", "SFMono-Regular", monospace` | 合同编号、金额、日期、百分比 |

不引入新的在线字体依赖。
标题通过字重和间距建立层级，不使用装饰性字体。

### 5.2 字号

| Token | 字号/行高 | 用途 |
|---|---|---|
| `text-xs` | 12/20 | 辅助信息、数据时间 |
| `text-sm` | 14/22 | 表格、表单、正文 |
| `text-md` | 16/24 | 区块标题、重要标签 |
| `text-lg` | 20/28 | 页面标题 |
| `text-xl` | 28/36 | 核心金额或关键比率 |

32px 以上字号只允许用于极少数单一主指标。
同一屏不得出现六个等权重大数字。

### 5.3 数字格式

- 金额统一显示为“万元”，保留两位小数，并使用千位分隔。
- 百分比保留一位小数。
- 计数显示整数。
- 负数使用减号，不用括号或仅靠红色表达。
- 未知金额显示“待核验”，不得显示为 `0.00`。
- 编号不截断关键后缀；空间不足时提供完整 Tooltip 和可复制操作。

## 6. 间距、圆角与层次

### 6.1 间距

全站只使用以下间距：

`4px、8px、12px、16px、24px、32px`

- 控件内部：8px 或 12px。
- 同组元素：8px 或 12px。
- 区块之间：16px 或 24px。
- 页面主要章节：24px 或 32px。

### 6.2 圆角与阴影

- 输入和按钮：4px。
- 卡片和表格容器：8px。
- 状态标签可以使用胶囊形，普通卡片禁止使用超大圆角。
- 常规内容优先用边框和留白分组。
- 只有浮层、抽屉和悬浮工具使用明显阴影。
- 页面卡片不使用发光、渐变描边或叠加阴影。

## 7. 页面模板

### 7.1 驾驶舱

顺序固定为：

1. 页面标题、数据时间和单位。
2. 四个已确认的本地视图切换。
3. 下一步行动。
4. 当前视图的关键指标。
5. 当前视图需要的结构和趋势。
6. 最近变化或重点对象列表。

驾驶舱保留单路由 `/dashboard/overview/index`，使用 `?view=all|management|projects|finance` 保存视图。
只加载当前视图需要的图表和数据。

### 7.2 列表页

- 使用 `Page` + `useVbenVxeGrid`。
- 搜索表单最多展示四个高频条件，其他条件进入“更多筛选”。
- 当前筛选以可移除标签显示。
- 工具栏左侧显示结果数量，右侧显示真实可用的批量操作。
- 对象编号或名称使用显式链接。
- 行高目标为 44px，表头目标为 40px。
- 单击行可以选中，不能把整行点击作为唯一详情入口。
- 列宽、固定列和溢出策略在模块级配置中统一。
- 不在 Vben 表格外重复包裹无标题、无语义的 `a-card`。

### 7.3 详情页

页首顺序固定为：

1. 返回来源。
2. 对象名称、编号和状态。
3. 数据来源、核验时间和页面操作。
4. 履约链。
5. 主要信息和侧边摘要。

详情页采用 12 列栅格：

- 主要内容 8 列。
- 财务、风险、待办和关联对象 4 列。
- 1024px 以下改为单列，侧边摘要移动到主要内容之后。

同一详情页最多保留一组页面内分段导航。
分段名称使用“基本信息、履约计划、财务、关联对象”等真实内容词。

### 7.4 履约链

履约链只用于能证明阶段关系的对象：

`合同签订 → 项目执行 → 阶段交付 → 已开票 → 已回款/已付款`

规则：

- 每个节点显示状态文字和最近更新时间。
- 未知节点显示“待核验”，不自动显示为未开始。
- 节点可进入关联对象时使用链接语义。
- 不计算没有业务依据的伪百分比。
- 移动端改为纵向时间线。
- 页面不存在完整链路时不渲染占位链。

## 8. 数据展示契约

### 8.1 标识

- 页面主标题使用对象名称。
- 合同号、项目号、发票号等作为可复制的辅助标识。
- 内部数据库 ID 不直接作为用户主标识。
- 外部编号映射未核验时显示“映射待核验”和时间。

### 8.2 金额

- API 字段或 Schema 必须明确单位。
- UI 不通过列标题猜测单位。
- 同一页面不得混用元和万元。
- 汇总和明细必须使用同一口径，并提供数据时间。

### 8.3 状态

后端原始值必须先映射到中文状态字典，不直接显示 `active`、`signed` 等原始值。

| 语义 | 示例 | 颜色 |
|---|---|---|
| 中性 | 草稿、待处理、未评估、待核验 | 中性灰 |
| 进行 | 执行中、部分开票、部分回款 | 调度蓝 |
| 完成 | 已完成、已开票、已回款、已付款 | 验收绿 |
| 待办 | 待交付、待回款、待付款 | 预警琥珀 |
| 异常 | 逾期、失败、已终止 | 故障红 |

一个原始状态只能映射到一个中文标签。
“进行中”和“执行中”不得在同一业务对象中混用。

### 8.4 风险与评分

- 风险只使用“未评估、低、中、高”四档。
- 风险标签必须能查看判断因子、规则版本和评估时间。
- 未运行风险检查时显示“未评估”，不能显示“低风险”。
- “综合评级”“优秀”“健康”等结论只有在口径、权重和适用范围可见时才能使用。
- 自动规则结果必须标记为“系统评估”，不能伪装成人工结论。

### 8.5 未知值

| 情况 | 显示 |
|---|---|
| 数据字段不存在 | `待补充` |
| 数据存在但未核验 | `待核验` |
| 不适用 | `不适用` |
| 已确认数值为零 | `0` 或 `0.00` |
| 请求失败 | 错误状态，不显示空值 |

## 9. 组件规则

优先使用 Vben 和 Ant Design Vue 已有组件：

| 场景 | 组件 |
|---|---|
| 页面壳层 | `Page` |
| 列表 | `useVbenVxeGrid` |
| 搜索表单 | `Vben Form` |
| 加载/错误/空态 | `StateBlock` |
| 结构化字段 | `a-descriptions` |
| 状态 | `a-tag`、`a-badge` |
| 时间关系 | `a-timeline` |
| 页面内视图 | `a-tabs` 或分段控件 |
| 图表 | `EchartsUI` + `useEcharts` |
| 次级编辑 | `Vben Drawer` |
| 关键确认 | `Vben Modal` |

不要为了“统一”把所有内容放入 `a-card`。
卡片必须有明确标题、对象边界或交互职责。

VxeTable、ECharts、API 和后端菜单的用法继续遵守 [Vben 框架开发规则](./vben-framework-rules.md)。

## 10. 文案与操作

- 使用用户能识别的对象和动作名称。
- 按钮使用明确动词，例如“保存修改”“新增发票”“导出当前结果”。
- 同一动作在按钮、确认框、成功提示中使用同一名称。
- 没有实现的操作不显示，不用“开发中”占位。
- 错误说明发生了什么、影响什么、用户下一步怎么做。
- 空态说明当前为什么没有结果，并提供可执行入口。
- 页面不使用 Emoji 作为状态、图标或业务含义。

## 11. 交互与动效

- 动效只用于状态变化、层级切换和操作反馈。
- 页面切换采用 120ms 至 180ms 的淡入或位移，不组合多个效果。
- 骨架屏只模拟实际布局，不播放持续闪烁。
- 数字不默认滚动计数。
- 图表不播放长入场动画。
- 遵守 `prefers-reduced-motion`，关闭非必要动效。
- 禁止为了“高级感”增加背景粒子、自动轮播和持续脉冲。

## 12. 可访问性

- 普通文字对比度不低于 4.5:1。
- 大号粗体文字对比度不低于 3:1。
- 组件边界、图标和非文本状态对比度不低于 3:1。
- 所有可点击元素使用链接或按钮语义。
- 键盘焦点环至少 2px，不能只靠颜色变化。
- 目标控件最小可点击区域为 40×40px。
- 表格必须提供显式操作入口，不能只依赖双击或悬停。
- 页面提供跳到主内容的 skip link。
- 图表必须提供文本摘要或可访问数据表。
- 颜色状态同时提供文字或图标。
- 1440px、1024px、768px 和移动端均不得横向溢出。

## 13. 性能预算

阶段 3 纵向切片的第一阶段预算：

| 指标 | 预算 |
|---|---:|
| 初始 JavaScript 传输量 | 不高于 2.0MB |
| 单个业务路由压缩后 Chunk | 不高于 500KB |
| 登录到首个可用页面的请求数 | 不高于 50 |
| 本地 Docker 冷启动可用时间 | 不高于 8 秒 |
| 本地 Docker 热路径可用时间 | 不高于 3 秒 |
| 页面交互导致的重复请求 | 0 |

驾驶舱图表按当前视图懒加载。
详情页重型模块按用户展开或滚动到可见区域后加载。
性能预算未通过时，不用骨架屏掩盖实际阻塞。

## 14. 页面状态清单

每个新增或重做页面必须覆盖：

- 加载。
- 空数据。
- 错误与重试。
- 成功反馈。
- 焦点。
- 禁用。
- 无权限。
- 数据未核验。
- 构建过期。
- 深色模式。
- 窄屏。

`StateBlock` 负责页面级加载、错误和空态。
图表和关联对象区块负责自己的局部空态。

## 15. 自动检查与验收

### 15.1 静态检查

- 业务组件不得新增十六进制颜色。
- 状态原始值不得直接渲染。
- 页面路由只引用规范路由名。
- 可点击 `div` 必须改为链接或按钮。
- VxeTable、ECharts 和 API 用法符合项目框架规则。

### 15.2 浏览器验收

- 使用真实浏览器完成菜单、列表、详情、关联对象和返回链路。
- 捕获 `console`、`pageerror`、`requestfailed` 和 HTTP 失败。
- 验证键盘路径、焦点顺序、对比度和屏幕阅读器名称。
- 验证 1440px、1024px、768px 和移动端。
- 验证浅色、深色和减少动效模式。
- 验证部署 SHA 与目标提交一致。

### 15.3 完成标准

- 主要任务不经过隐藏旧页面。
- WCAG 2.1 AA 对比度通过。
- 所有状态与金额口径可追溯。
- 性能预算通过。
- 控制台错误、页面错误、失败请求和 HTTP 失败均为 0。
- 纵向切片通过用户首击测试后，才允许复制到其他模块。
