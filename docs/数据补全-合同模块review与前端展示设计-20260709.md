## 2. 前端改造方案

### 2.1 数据接入
```ts
const clauses = computed(() => detail.value?.clauses ?? [])
```

### 2.2 新增区块（`.panel` 风格，置于「交付物」之后）
- **标题**：违约 / 罚款条款
- **分组**：按 `clause_category` 聚合成 7 必含 + 2 可选 组，每组一个子标题。
- **每条款卡片**展示字段：
  - `trigger_type`（触发情形）：逾期交付 / 验收不通过 / 保密违约 / 解除合同 / 其他
  - `rate_pct`：如 `违约金 1% / 天`，基准 `ld_base`（技术服务报酬）
  - `threshold_days`：逾期解除阈值（N 日）
  - `refund_full`：徽标「退还全部款项」（用 `--status-danger` / `--finance-unreceived` 语义色）
  - `clause_text`：原文片段（折叠/展开，满足审计留痕）
- **空态**：`clauses.length === 0` 时显示「暂无条款数据」（`state-block` empty 风格）。

### 2.3 视觉规范（ardot）
- 区块外壳使用 `.panel`（非 `a-card`），与全局统一方向一致。
- 比例/金额用 `finance-tokens.css` 语义色；`退还全部款项` 用 `--status-danger`。
- 徽标（badge）统一圆角、低密度，避免与既有 `a-card` 视觉冲突（过渡期允许并存，统一见 #1）。

### 2.4 验收标准
- [ ] `GET /api/contracts/{id}` 返回 `clauses` 且前端渲染。
- [ ] 条款按 9 类分组展示，含触发情形 / 比例 / 阈值 / 退还徽标 / 原文。
- [ ] 无条款合同时显示空态，不报错。
- [ ] 区块使用 `.panel` 风格，语义色取自 `finance-tokens.css`。
- [ ] 本地 Docker 启动 + Playwright 冒烟验证通过（见 #3.⑤）。
