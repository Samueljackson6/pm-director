### 1.1 核心铁律

#### 飞书表格增量维护铁律（2026-06-12 强制）

**违反后果**：数据永久丢失，无法恢复

**核心原则**：
- ❌ 禁止直接覆盖任何字段
- ✅ 所有更新必须先读取、再追加、最后更新

**操作流程**：
```
步骤1：读取当前"最后沟通记录"
步骤2：将当前内容追加到"历史沟通记录"（换行分隔）
步骤3：更新"最后沟通记录"为新内容
```

**适用字段**：
- 最后沟通记录 / 历史沟通记录
- 回款进度 / 回款进度历史
- 待办事项 / 待办事项历史
- 备注 / 备注历史

**时间格式强制规范**：
- 标准：`YYYY-MM-DD`
- 回款：`YYYY-MM-DD：金额元`

---

#### 统计数据铁律（2026-06-29 强制）

**违反后果**：数据错误，误导决策

**核心原则**：
- ❌ 禁止写死任何统计数据
- ✅ 必须从飞书表格实时读取并计算
- ✅ 统计口径：2026年已开票/已回款 = 所有年份合同在2026年实际发生的金额

**错误案例**：
- HTML显示"2026年已开票 235.60万元"，实际应为**361.74万元**
- 原因：写死数据，未从飞书表格实时计算

**正确做法**：
1. 使用飞书API读取所有项目数据
2. 解析每个项目的"开票进度"和"回款进度"字段
3. 提取2026年的开票/回款记录
4. 汇总计算

### 3.2 核心表结构

#### contracts（合同表）

```sql
CREATE TABLE contracts (
    contract_id TEXT PRIMARY KEY,      -- 项目编号（ZH02-YYYYMMNNN）
    project_name TEXT,                  -- 项目名称
    project_type TEXT,                  -- 项目类型（科研类/服务类）
    contract_amount REAL,               -- 合同金额（万元）
    party_a TEXT,                       -- 甲方
    party_b TEXT,                       -- 乙方
    sign_date DATE,                     -- 签订日期
    project_leader TEXT,                -- 项目负责人
    service_content TEXT,               -- 服务内容
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);
```

- 总金额：2328.64万元

---

#### finance_records（财务记录表）⭐ 核心

```sql
CREATE TABLE finance_records (
    record_id INTEGER PRIMARY KEY,
    project_id TEXT,                    -- 项目编号
    project_name TEXT,
    customer TEXT,
    
    -- 合同信息
    contract_amount REAL,
    contract_date DATE,
    
    -- 开票信息
    invoice_total REAL,                 -- 已开票总额（万元）
    invoice_unbilled REAL,              -- 未开票金额（万元）
    invoice_history TEXT,               -- 开票历史记录
    
    -- 回款信息
    payment_total REAL,                 -- 已回款总额（万元）
    payment_unreceived REAL,            -- 未回款金额（万元）
    payment_history TEXT,               -- 回款历史记录
    
    -- 供应商财务
    subcontractor TEXT,
    sub_invoice_total REAL,
    sub_payment_total REAL,
    
    -- 元数据
    batch_id TEXT,                      -- 导入批次（如"2026-W26"）
    import_time TIMESTAMP,
    
    FOREIGN KEY (project_id) REFERENCES contracts(contract_id)
);
```

**数据统计**：

- 供应商付款：287.03万元

**核心特点**：
- ✅ 每周追加新记录，不覆盖历史
- ✅ 通过batch_id标识导入批次
- ✅ 支持供应商财务管理
