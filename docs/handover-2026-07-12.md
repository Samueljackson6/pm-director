## 二、未完成的核心任务

### 1. 甘特图条块不显示 bug 🐛

**现象**: 科研类合同详情页的阶段进度甘特图，canvas 渲染了（1126×320，27KB 内容），但条块（彩色矩形）可能不显示。

**诊断**:
- 阶段数据正常（5 个阶段，都有完整 start_time/end_time）
- Canvas 已渲染，无降级清单，无 console 报错
- **怀疑原因**: `stage-gantt.vue` 第 207 行 `bandHeight` 计算在最后一个阶段时可能为 0

**修复方案**（已定位，待实施）:
```javascript
// stage-gantt.vue 第 205-210 行，加入防御性代码
const topPx = api.coord([startTs, catLabel]);
let bandHeight = 40; // 默认值
if (catIdx < yLabels.length - 1) {
  const next = yLabels[catIdx + 1];
  const bottomPx = api.coord([startTs, next]);
  bandHeight = Math.abs((bottomPx?.[1] ?? 100) - (topPx?.[1] ?? 0));
}
bandHeight = Math.max(bandHeight, 24); // 确保最小高度
```

**验证方法**: 修复后重新构建，刷新科研类合同（ZH02-202604023），看甘特图是否出现彩色条块。

## 四、关键记忆点（防重犯）

### 1. project_type 实际取值 ⚠️
- DB 里只有 `'科研类'`(10个) 和 `'服务类'`(32个)
- **没有"技术服务类"**（数据治理文档用"技术服务类"作描述性称呼，但 DB 存的是"服务类"）
- **前端 v-if/computed 判断必须用 `=== '服务类'`**，不能用"技术服务类"

### 2. 后端 update_contract early-return 陷阱 ⚠️
- `PUT /{id}` 当 payload 只含 stages、没有 updatable 字段时，`if not fields: return` 在处理 stages 之前提前返回
- **测试阶段更新时务必带一个 updatable 字段**（如 project_name），否则 stages 静默不更新
- 真实前端 saveEdit 发完整 editForm 不触发，但 stages-only 更新会失效

### 3. 甘特图渲染关键点
- 使用 ECharts `type: 'custom'` + `renderItem`
- **`api.coord([timeValue, yValue])` 的 yValue 必须传类别标签字符串**（如 `'研究阶段5.1'`），而非数字索引
- `bandHeight` 通过 `api.coord` 的 y 坐标差手动计算，注意边界情况（最后一个阶段）

### 4. 数据相关判断必须用 API/DB 验证实际取值
- 不能信文档描述（文档说"技术服务类"，实际是"服务类"）
- 先查实际数据再写 v-if 条件
