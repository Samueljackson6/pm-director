## 六、根因分析

### 6.1 数据提取问题

**问题**: 服务类合同的 service_content 全部为空

**根因分析**:
1. OCR 文本中包含服务内容（如合同原文的"服务内容"章节）
2. 数据提取脚本 `contract_intelligent_recognizer.py` 可能未正确提取服务类合同的服务内容字段
3. 或者提取后的 JSON 数据导入时丢失

**建议修复**:
```python
# 需要检查 ServiceProjectParser 的提取逻辑
# 特别关注 service_content 字段的正则匹配
```

### 6.2 付款金额不匹配问题

**问题**: 34个合同的付款总额与合同金额不符

**根因分析**:
1. 付款计划来源于 OCR 提取的"项目支付计划"章节
2. 合同金额来源于合同首页的金额信息
3. 可能存在：
   - OCR 识别金额错误
   - 多阶段付款提取遗漏或重复
   - 单位换算错误（元→万元）

**建议修复**:
- 重新核对 OCR 文本中的付款计划金额
- 验证金额累加逻辑
- 检查去重逻辑

### 6.3 孤儿记录问题

**问题**: 存在关联到不存在合同的 stages/payments/deliverables

**根因分析**:
1. 数据导入顺序问题：先导入关联表，后导入主表
2. 或主表导入失败但关联表成功
3. 外键约束未启用或被忽略

**建议修复**:
```sql
-- 清理孤儿记录
DELETE FROM stages WHERE contract_id NOT IN (SELECT contract_id FROM contracts);
DELETE FROM payments WHERE contract_id NOT IN (SELECT contract_id FROM contracts);
DELETE FROM deliverables WHERE contract_id NOT IN (SELECT contract_id FROM contracts);
```
