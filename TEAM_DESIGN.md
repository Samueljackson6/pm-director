## 二、项目类型区分

### 2.1 内部项目 vs 外部项目

| 维度 | 内部项目 | 外部项目 |
|------|----------|----------|
| **来源** | 电力大管家、搞钱管家等Agent产生的项目 | 你交给团队管理的公司项目 |
| **标识** | `internal-*` | `external-*` |
| **工作空间** | `workspace/pm-director/projects/internal/` | `workspace/pm-director/projects/external/` |
| **汇报频率** | 每周汇总 | 每日简报 + 每周详报 |
| **交付标准** | Agent自验收 + 复盘 | 人工验收 + 客户确认 |
| **文档要求** | Markdown为主 | MD + Word + Excel |
| **飞书同步** | 可选 | 强制 |
| **预警级别** | 标准预警 | 强化预警（电话可选） |

### 2.2 项目分类标识

```
项目ID格式：{类型}-{团队}-{序号}-{日期}

示例：
- internal-power-001-20260413  （电力大管家内部项目）
- internal-lobster-002-20260415（搞钱管家内部项目）
- external-clientA-001-20260413（客户A外部项目）
- external-product-002-20260420（产品开发项目）
```

### 2.3 工作空间结构

```
~/.openclaw/workspace/pm-director/
├── projects/
│   ├── internal/                    # 内部项目
│   │   ├── power/                   # 电力大管家项目
│   │   ├── lobster/                 # 搞钱管家项目
│   │   ├── jrzj/                    # JRZJ项目
│   │   └── system/                  # 系统优化项目
│   │
│   └── external/                    # 外部项目
│       ├── clients/                 # 客户项目
│       ├── products/                # 产品开发
│       └── operations/              # 运营项目
│
├── templates/
│   ├── internal/                    # 内部项目模板
│   └── external/                    # 外部项目模板
│
├── reports/
│   ├── internal/                    # 内部项目报告
│   └── external/                    # 外部项目报告
│
```

| 属性 | 配置 |

| **模型** | jdcloud/GLM-5 |

### 3.7 交付管控师 (delivery_agent) ⭐核心新增

| 属性 | 配置 |
|------|------|
| **模型** | baidu-qianfan/qianfan-code-latest |
| **工作空间** | 共享 |
| **身份** | 交付物守门员 |

**核心能力**：
- **交付标准制定**：为每个项目/里程碑定义交付物清单、验收标准、格式要求
- **交付物验收**：自动检查完整性、格式、内容质量，生成验收报告
- **版本管理**：统一管理版本，自动命名、归档、保留历史版本
- **逾期管控**：跟踪交付进度，提前预警，逾期自动升级
- **多渠道同步**：验收通过的交付物自动同步到飞书、本地、共享空间
- **整改跟踪**：对不合格交付物生成整改意见，跟踪整改进度

**交付管控标准流程**：
```
planner制定交付计划
    ↓
delivery定义验收标准
    ↓
责任人提交交付物
    ↓
delivery自动验收
    ├─ 合格 → 归档同步（飞书+本地+Word+Excel）
    └─ 不合格 → 生成整改意见 → 重新提交
```

**逾期预警机制**：

| 时间节点 | 动作 | 通知渠道 |
|----------|------|----------|
| 交付前7天 | 首次提醒，确认准备情况 | 对话 + 飞书@ |
| 交付前3天 | 二次提醒，检查进度 | 对话 + 飞书@ |
| 交付前1天 | 最终提醒，预警逾期风险 | 对话 + 飞书@ |
| 交付当天 | 自动验收，生成验收报告 | 对话 + 飞书文档 |
| 逾期1天 | 升级预警，通知你介入 | 对话 + 飞书@ + 邮件 |

**输出物**：
- 交付物清单
- 验收报告
- 整改通知
- 版本历史记录

## 四、多渠道交互

### 4.1 日常对话交互

直接与 pm-director 对话：

```
示例对话：
- "给我看看电力大管家项目的最新进度"
- "把项目周报改成PPT格式发给我"
- "这个交付物不合格，重新修改"
- "创建一个外部项目：XX客户官网开发"
```

### 4.2 飞书文档深度集成

**自动同步能力**：
- 项目启动时，自动在飞书创建项目专属文件夹
- 所有文档自动同步到飞书对应位置
- 项目进度、风险实时更新到飞书项目仪表盘
- 里程碑到期、交付物提交、风险发生时，自动@你提醒

**双向交互能力**：
- 你可以直接在飞书文档中编辑、评论、修改
- pm-director 自动扫描飞书文档的修改和评论
- 自动提取任务和反馈
- 自动生成回复和行动项

**配置示例**：
```json
{
  "plugins": {
    "feishu": {
      "enabled": true,
      "appId": "你的飞书应用ID",
      "appSecret": "你的飞书应用密钥",
      "defaultFolderId": "飞书项目根文件夹ID",
      "autoCreateFolders": true,
      "autoSyncDocuments": true,
      "watchComments": true,
      "syncFormats": ["markdown", "docx", "xlsx"]
    }
  }
}
```

### 4.3 Word/Excel 自动生成

**自动生成能力**：
- 所有 Markdown 文档自动同步生成 Word (.docx)
- 所有表格类内容自动生成 Excel (.xlsx)
- 生成的文件自动保存到本地和飞书

**文件上传交互**：
- 你可以直接上传 Word/Excel 文件
- 自动解析内容，提取项目信息
- 自动更新项目计划、进度、风险登记册

## 五、工作流程

### 5.1 标准项目流程

```
你下达项目指令 → pm-director识别项目类型（内部/外部）
    ↓
initiation_agent 做立项分析 → delivery_agent 定义立项交付物标准
    ↓
planner_agent 做项目规划 → delivery_agent 定义各里程碑交付物标准
    ↓
execution_agent 跟踪执行 → 提交交付物给 delivery_agent
    ↓
delivery_agent 自动验收
    ├─ 合格 → 归档同步（飞书+Word+Excel）
    └─ 不合格 → 返回整改
    ↓
risk_agent 监控风险 → document_agent 生成汇报材料
    ↓
review_agent 复盘优化 → pm-director 汇总汇报给你
```

### 5.2 内部项目流程（简化）

```
Agent产生项目需求 → pm-director登记为内部项目
    ↓
planner_agent 快速规划 → execution_agent 跟踪
    ↓
Agent自验收 → delivery_agent 复核
    ↓
review_agent 复盘 → 更新知识库
```

**内部项目特点**：
- 流程简化，减少审批环节
- Agent 自验收为主，delivery 复核为辅
- 每周汇总汇报即可
- Markdown 为主，无需多格式输出

### 5.3 外部项目流程（完整）

```
你下达外部项目指令 → pm-director创建外部项目
    ↓
完整流程（立项→规划→执行→交付→收尾→复盘）
    ↓
每日简报 + 每周详报
    ↓
多格式输出（MD+Word+Excel）+ 飞书强制同步
    ↓
人工验收 + 客户确认
    ↓
强化预警（电话可选）
```

**外部项目特点**：
- 完整流程，每个环节都有产出
- 每日简报 + 每周详报
- 多格式输出 + 飞书强制同步
- 人工验收 + 客户确认
- 强化预警机制
