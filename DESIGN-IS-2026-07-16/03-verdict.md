# 审计结论：REDESIGN

## 结论

本轮必须重设计真实 Vben 主应用的信息架构、驾驶舱和页面模板；不是替换技术栈，也不是继续维护被否决的 Phase3Review Demo。

## 保留

- Vben `BasicLayout`、后端动态菜单、单应用路由体系。
- `Page`、`StateBlock`、`useVbenVxeGrid`、Vben Form/Modal/Drawer。
- `EchartsUI/useEcharts` 的图表挂载与响应式机制。
- 合同、项目、阶段、交付物、文件、发票、回款、供应商的现有只读 API 与聚合关系中经核验可用的部分。
- 驾驶舱单次聚合请求和查询参数保存视角的技术机制。

## 必须重构

1. 菜单：恢复项目履约正式入口，合并供应商管理与供应商财务的组织层级，消除 Legacy 冲突和双源码契约。
2. 驾驶舱：从 KPI 墙改为“角色任务工作台 + 数据可信度中心”，先行动、再风险/到期、再可信指标和趋势。
3. 资金链：客户应收与供应商应付分轨；供应商数据源未建立时提供诚实空态，而不是借用历史客户回款。
4. 页面：统一 `Page` 根壳、筛选、表格、详情头、事实状态、弹层和返回上下文。
5. 内容：所有指标提供口径、来源、覆盖范围、核验状态和截止时间；未知值不自动转为 0。
6. 交互：显式详情入口、键盘语义、模块级状态、移动任务卡和 reduced-motion。

## 视觉方向

采用“电网白图 / Electric Paper”方向：浅灰白画布、清晰墨色、极少量电蓝与青绿高光、细网格和关系线只用于结构提示。科技感来自数据关系、精准排版、编号字体、微动效和事实状态，而不是大块暗色、霓虹渐变或装饰粒子。

跨页标志性元素是“事实链”：在合同、项目、财务和供应商对象头部，以紧凑刻度表达原件、OCR、候选值、人工核验和业务事件所处层级。它必须可点击查看来源，并在数据缺失时直接给出补齐动作。

## 通过门槛

- 菜单和所有核心路由可达，详情返回保持来源上下文。
- 页面不再混用客户应收和供应商应付数据。
- 金额单位、核验状态、来源和数据时间可解释。
- 390、768、1024、1440 宽度下核心任务可完成且不隐藏关键事实。
- 键盘、对比度、焦点、图表摘要和 reduced-motion 通过专项验收。
- 生产代码的每项变更均有 RED→GREEN 测试证据和浏览器 QA 产物。
