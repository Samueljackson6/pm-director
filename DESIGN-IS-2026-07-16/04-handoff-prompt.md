# 阶段 3 重设计实施交接提示

请在 PM Director 当前 Vben 主应用中执行一次以业务真相为核心的系统性重设计。

## 目标信息架构

```text
综合工作台
└─ 经营与任务驾驶舱

合同履约
├─ 合同台账
└─ 项目台账

客户应收
├─ 销项发票
└─ 客户回款

供应链
├─ 供应商档案
├─ 供应商收票
└─ 供应商付款
```

四个一级组分别是经营工作台、合同履约、客户应收和供应链。详情页作为隐藏子路由归属相应对象域。数据核验不先扩张为第五个一级菜单，首期作为驾驶舱角色视角、合同事实链和各对象页的待核验队列落地；当审核任务实体与权限模型建立后再评估独立工作台。

## 驾驶舱

默认顺序为：我的待办与缺口 → 风险/到期 → 数据可信度 → 角色摘要 → 最近变化。角色视角建议为综合、经营、项目、财务、数据核验。每个视角必须改变决策内容和动作，而不只是重排同一组 KPI。

指标必须采用：`值 + 状态 + 截止时间 + 来源 + 已核验/总数 + 解释 + 动作`。供应商付款数据源未建立、风险待核验、单批次无法形成趋势等情况必须直接表达。

## 页面模板

- 列表：`Page` + 紧凑任务摘要 + Vben Grid 查询/工具栏 + 模块状态 + 显式详情操作。
- 详情：返回上下文 + 对象标题/编号 + 事实链 + 关键义务/金额条 + 主任务区 + 关联对象侧栏。
- 核验：Vben Drawer 展示原文、候选值、差异、采用值、理由和审计信息。
- 写入：Vben Modal，提交锁、明确校验和成功后刷新。
- 移动：列表降级为保留金额、状态、期限和动作的任务卡；不得简单隐藏关键列。

## 视觉与动效

- 浅色画布和白色层面，使用 Vben Token 承载主题。
- 电蓝仅用于主动作和关系线，青绿表示已核验，琥珀/红只表示风险。
- 合同编号、金额、日期使用统一数字字体栈和 tabular numbers。
- 动效 140–220ms，仅用于状态反馈、抽屉、选中关系和首屏轻量分段出现；支持 reduced-motion。
- 禁止大块暗色、装饰性霓虹、粒子背景、卡片墙、Emoji 图标和可点击 `div/span`。

## 实施纪律

1. 先测试后代码，保存每次失败和通过的证据。
2. 先处理路由真相、基础 Token、语义链接和页面壳，再按业务域并行改造。
3. `contracts/detail.vue` 与 `suppliers/detail/index.vue` 超长，必须先用回归测试锁定行为并拆分。
4. 不改业务数据库，不执行自动匹配，不清洗历史数据，不提交 Git。
5. 所有页面由真实浏览器完成桌面、移动、键盘、错误态、空态和返回上下文验收。
