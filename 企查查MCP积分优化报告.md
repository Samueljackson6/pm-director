## 📋 查询内容说明

### ✅ 已查询（低积分）

#### 1. 工商信息 (get_company_registration_info)
**为什么查询**: 企业基本身份识别，必需信息
**积分消耗**: 低（~1 积分）
**包含内容**:
- 企业名称、统一社会信用代码
- 法定代表人、注册资本、成立日期
- 登记状态、注册地址
- 参保人数、经营范围

#### 2. 企业简介 (get_company_profile)
**为什么查询**: 了解企业业务定位
**积分消耗**: 低（~1 积分）
**包含内容**:
- 企业简介、主营业务
- 企查查行业分类

#### 3. 风险扫描 (get_company_risk_scan)
**为什么查询**: 企业风险初筛，重要指标
**积分消耗**: 中等（~2 积分）
**包含内容**:
- 35项风险因子聚合扫描
- 有记录/无记录统计
- 风险因子明细（不深入查询）

#### 4. 软件著作权 (get_software_copyright_info)
**为什么查询**: 技术实力评估
**积分消耗**: 低（~1 积分）
**包含内容**:
- 软件著作权数量和列表

**积分消耗**: 1-10 积分不等
**查询时间**: < 10秒

## 📝 积分优化原则

### ✅ 优化原则

1. **必需优先**: 先查询必需的低积分接口
2. **避免重复**: 避免查询内容重叠的接口
3. **聚合优先**: 使用聚合扫描代替逐一查询
4. **按需深入**: 先扫面，再根据需要深入查询
5. **成本控制**: 设定积分预算，超预算暂停

### ❌ 避免做法

1. **全量查询**: 不分需求查询所有接口
2. **重复查询**: 查询内容重叠的多个接口
3. **明细过度**: 不需要时深入查询明细数据
4. **无预算查询**: 不设定积分上限
