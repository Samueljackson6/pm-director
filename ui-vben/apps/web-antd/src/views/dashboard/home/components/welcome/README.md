# 欢迎组件 (Workbench Welcome)

## 组件概述

欢迎组件是首页的核心展示组件，用于向用户展示个性化的问候信息、实时天气和日期信息。组件采用动态背景设计，根据实际天气状况自动调整视觉效果。

## 核心特性

### 1. 智能问候系统

- **时段问候**：根据当前时间自动显示不同问候语
  - 凌晨好（0-6点）
  - 早上好（6-9点）
  - 上午好（9-12点）
  - 中午好（12-14点）
  - 下午好（14-18点）
  - 晚上好（18-22点）
  - 夜深了（22-24点）
- **用户识别**：显示用户真实姓名或用户名
- **自定义提示**：支持配置个性化提示语

### 2. 精准天气系统

- **地理定位**：
  - 优先使用浏览器Geolocation API获取精确经纬度
  - 超时保护：5秒自动降级
- **城市获取**：
  - 统一使用和风天气GeoAPI（City Lookup）
  - 如果有经纬度，使用经纬度查询城市
  - 如果没有经纬度，使用IP自动定位
  - 自动去除城市名称后缀（市、自治州等）
- **天气信息**：
  - 城市名称（自动去除后缀）
  - 实时温度
  - 天气状况
  - 湿度
  - 风向
- **天气图标**：支持和风天气全部图标代码映射

### 3. 固定背景 + 天气装饰

- **统一背景**：固定蓝色渐变 `#4facfe → #00f2fe`（与原型一致）
- **天气装饰**：根据实际天气在右侧天气框内显示不同装饰

#### 晴天 (Clear)

- 装饰效果：脉动太阳（右上角）
- 动画：脉动缩放 + 旋转光晕

#### 多云 (Cloudy)

- 装饰效果：2朵小云
- 动画：轻柔飘动

#### 雨天 (Rainy)

- 装饰效果：3条小雨滴
- 动画：从上落下

#### 雪天 (Snowy)

- 装饰效果：3个小雪花
- 动画：旋转飘落

#### 雾霾 (Foggy)

- 装饰效果：朦胧雾气
- 动画：左右移动

### 4. 农历日历系统

- **完整农历转换**（1900-2100年）
- **显示内容**：
  - 天干地支年份（如：甲子年）
  - 生肖（如：鼠年）
  - 农历月份（支持闰月）
  - 农历日期（如：正月初一）
- **显示格式**：`甲子鼠年 正月初一`

### 5. 日期显示

- **公历日期**：`2026-01-09 星期五`
- **农历日期**：`甲子鼠年 正月初一`

## 配置说明

### Props

```typescript
interface Props {
  greeting?: string; // 自定义提示语
  showWeather?: boolean; // 是否显示天气
  weatherApiKey?: string; // 和风天气API Key
}
```

### 默认值

```typescript
{
  greeting: '欢迎回来，开始您的工作吧！',
  showWeather: true,
  weatherApiKey: ''
}
```

## 使用方式

### 1. 基础使用（模拟数据）

```vue
<template>
  <WorkbenchWelcome />
</template>
```

### 2. 配置天气API

```vue
<template>
  <WorkbenchWelcome
    greeting="今天也要加油哦！"
    :show-weather="true"
    weather-api-key="your-qweather-api-key"
  />
</template>
```

**说明**：

- `weather-api-key`：必需，用于获取城市和天气数据

### 3. 在首页设计器中使用

1. 进入"系统管理 → 首页管理"
2. 点击"设计"按钮
3. 从左侧组件面板拖拽"欢迎组件"到画布
4. 在右侧配置面板中设置：
   - 提示语
   - 是否显示天气
   - 和风天气API Key
5. 保存布局

## API配置

### 和风天气API Key（必需）

用于获取城市信息和天气数据。

1. 访问：https://dev.qweather.com/
2. 注册账号
3. 创建应用（选择免费开发版）
4. 获取API Key

**配置方式**：在首页设计器中配置 `weatherApiKey` 属性

**API配额说明**：

- **免费版**：每天1,000次请求
- **付费版**：更高请求额度

**使用的API**：

- GeoAPI - City Lookup（城市查询）
  - 支持经纬度查询
  - 支持IP自动定位
- Weather API - Now（实时天气）

### 城市获取方案

统一使用和风天气GeoAPI：

1. ✅ **经纬度查询**（如果浏览器授权定位）
   - 使用浏览器Geolocation API获取经纬度
   - 通过经纬度精确查询城市
   - 准确度最高
2. ✅ **IP自动定位**（如果没有经纬度）
   - 使用和风天气的IP定位功能
   - 根据用户IP自动获取城市
3. ⚠️ **默认城市**（深圳，所有方案都失败时）

## 浏览器兼容性

### Geolocation API支持

- ✅ Chrome 5+
- ✅ Firefox 3.5+
- ✅ Safari 5+
- ✅ Edge 12+
- ✅ Opera 10.6+
- ❌ IE 9+（支持但体验较差）

### 重要提示

- **HTTPS要求**：Geolocation API只在HTTPS环境下工作
- **用户授权**：首次使用需要用户授权位置权限
- **降级处理**：如果浏览器不支持或用户拒绝，自动降级到IP定位

## 性能优化

### 1. 动画性能

- 使用CSS动画（GPU加速）
- 避免JavaScript频繁操作DOM
- 条件渲染减少不必要的DOM元素

### 2. API请求优化

- 地理位置请求有超时保护（5秒）
- 失败时自动降级到IP定位
- 建议添加缓存机制（避免频繁请求）

### 3. 内存优化

- 组件卸载时清理定时器
- 避免内存泄漏
- 合理使用computed和watch

## 故障排查

### 问题1：天气不显示

**可能原因**：

- API Key未配置或错误
- 网络请求失败
- API配额用完

**解决方案**：

1. 检查API Key是否正确
2. 查看浏览器控制台错误信息
3. 检查和风天气账号配额

### 问题2：城市名称不准确

**可能原因**：

- 用户拒绝地理位置权限，使用IP定位精度较低
- IP定位服务返回的城市信息不准确

**解决方案**：

1. 确保在HTTPS环境下，引导用户授权地理位置权限
2. 检查浏览器控制台，查看是否使用了经纬度定位还是IP定位
3. 如果IP定位不准确，可以手动在代码中设置默认城市

### 问题3：农历显示错误

**可能原因**：

- 日期超出支持范围（1900-2100）
- 农历数据表问题

**解决方案**：

1. 确认日期在1900-2100年范围内
2. 检查浏览器控制台错误
3. 如有bug请提交issue

## 最佳实践

### 1. API Key安全

- 不要将API Key硬编码在前端代码中
- 使用环境变量管理
- 考虑后端代理转发请求

### 2. 用户体验

- 首次加载时显示骨架屏
- 失败时显示友好的错误提示
- 提供重试按钮

### 3. 缓存策略

```typescript
// 建议添加缓存
const CACHE_KEY = 'weather_cache';
const CACHE_DURATION = 15 * 60 * 1000; // 15分钟

// 从缓存读取
const cachedData = localStorage.getItem(CACHE_KEY);
if (cachedData) {
  const { data, timestamp } = JSON.parse(cachedData);
  if (Date.now() - timestamp < CACHE_DURATION) {
    return data;
  }
}
```
