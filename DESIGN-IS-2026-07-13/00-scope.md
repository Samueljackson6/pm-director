# 设计审计范围

## 审计对象

- 运行地址：`http://localhost:18090/web/`
- 页面：综合看板、合同列表/详情、项目列表/详情、客户发票列表/详情、供应商列表/详情及全局壳层
- 代码：`ui-vben/apps/web-antd/src/`、`backend/routers/auth.py`、动态路由生成器
- 当前构建：本机 Docker 前端；该构建落后于工作区部分最新源码，因此同时区分“已运行设计”和“仅源码存在的设计”

## 主要用户与任务

主要用户是项目总监和项目管理人员。

主要任务是快速判断合同、项目、开票、回款、付款和风险状态，并从列表进入详情、查看关联对象后可靠返回原工作上下文。

## 约束

- 保留 Vue 3、Vben Admin 5、Ant Design Vue 和 FastAPI 技术栈；
- 中文界面与中文文档；
- 先设计、再实现，不在本审计内修改业务代码；
- 无障碍底线为 WCAG 2.1 AA；
- 数据单位、状态和风险标签必须能追溯到明确业务口径；
- 旧路径迁移必须有退役期限，不能长期保留双页面。

## 参考基线

- 项目规则：`docs/vben-framework-rules.md`、`docs/UI设计规范.md`
- Vben 本地官方文档：`ui-vben/docs/src/guide/in-depth/access.md`
- 当前页面截图：`docs/audits/evidence/2026-07-13/`
- 未指定竞品；以用户主要任务、Dieter Rams 十原则和 Vben 官方模式作为判断基线。

## 不在本次评分范围

- 远端 236 或公网环境；
- 移动端、深色模式和高对比度模式；
- 未实际渲染的上游 Vben 示例模块；
- 数据清洗结果本身；数据问题仅评估其对界面诚实性和可用性的影响。
